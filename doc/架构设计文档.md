# 斛生 客户端架构设计文档

## 1. 文档目标

本文档用于说明当前客户端的真实架构基线、目录结构、分层边界和平台适配策略。本文档只描述当前仓库仍然存在的代码，不再描述已经删除的识别、视频和历史记录模块。

## 2. 架构目标

当前架构围绕三个目标组织：

1. 对齐 `../shihu-web` 的真实后端能力
2. 保持跨平台页面结构和交互一致
3. 让页面、状态、仓储和适配层职责清晰，便于继续扩展

## 3. 分层设计

当前采用：

`feature-first + 分层解耦 + 可替换仓储`

### 3.1 App 层

职责：

- 应用入口
- 路由
- 全局主题
- 工作台壳层

当前目录：

- `lib/main.dart`
- `lib/app/app.dart`
- `lib/app/routes.dart`
- `lib/app/theme.dart`
- `lib/app/widgets/app_workspace_scaffold.dart`

### 3.2 Core 层

职责：

- 环境配置
- 常量
- 网络层
- 本地存储
- 平台适配工具

当前目录：

- `lib/core/config`
- `lib/core/constants`
- `lib/core/network`
- `lib/core/storage`
- `lib/core/utils`

补充说明：

- `ApiClient` 统一承接请求、错误归一化和 Cookie 处理
- `ServiceEndpointResolver` 只维护当前设备服务地址解析
- 不再保留视频服务端点和历史记录本地结构

### 3.3 Shared 层

职责：

- 当前版本通用模型
- 通用卡片、按钮、布局和背景组件
- 与页面无关的轻量动画能力

当前目录：

- `lib/shared/models`
- `lib/shared/widgets`

当前共享组件重点：

- `AppBackdrop`
- `CommonCard`
- `CommonButton`
- `AdaptiveWrapGrid`
- `ResponsiveInfoRow`
- `RevealOnMount`

### 3.4 Features 层

职责：

- 页面容器
- 页面状态
- 业务仓储
- 与具体页面直接相关的 UI 组件

当前目录：

- `lib/features/splash`
- `lib/features/auth`
- `lib/features/home`
- `lib/features/realtime`
- `lib/features/settings`
- `lib/features/about`

## 5. 状态与数据流

### 5.1 认证链路

`SplashPage -> AuthController -> AuthRepository -> RealAuthRepository / MockAuthRepository`

说明：

- 真实模式走 Spring Boot 会话接口
- 联调模式只在开发 / 测试环境且显式开启时生效

### 5.2 设备监控链路

`RealtimeDetectPage -> RealtimeDetectController -> DeviceStateRepository -> ApiClient`

说明：

- 控制器负责轮询、手动刷新和 LED 控制状态
- 仓储负责请求 `/api/status` 与 `/api/ops/led`

### 5.3 运维设置链路

`SettingsPage -> SettingsController / ServiceHealthRepository / DeviceStateRepository`

说明：

- 设置页不直接写请求逻辑
- 服务健康、设备状态和会话信息由各自 Provider 提供

## 6. 路由与导航

当前运行时只保留以下主路由：

- `SplashPage`
- `LoginPage`
- `HomePage`
- `RealtimeDetectPage`
- `SettingsPage`
- `AboutPage`

导航策略：

- 受保护页面统一受登录守卫控制
- 首页、主控台、设置页、系统总览页统一使用 `AppWorkspaceScaffold`
- 桌面端使用 `NavigationRail`
- 紧凑宽度下使用 `NavigationBar`
- `GoRouter` 通过认证状态刷新信号触发重定向，不在登录态变化时整套路由重建

## 7. 平台适配策略

- 页面业务逻辑尽量全平台复用
- 平台差异收敛到 `core/utils` 和存储 / 网络适配层
- Web 端认证请求依赖浏览器 Cookie
- 非 Web 端额外保存会话 Cookie
- 平台工程目录已覆盖 Android、iOS、macOS、Linux、Windows、OpenHarmony / 鸿蒙

## 8. 视觉与交互架构

当前视觉系统围绕统一工作台组织：

- `AppTheme` 统一颜色、圆角、阴影、导航和页面转场
- `AppBackdrop` 统一背景渐变、光斑和网格氛围
- `CommonCard` 统一卡片层级
- `RevealOnMount` 统一首屏区块入场节奏
- 工作台主色统一为青绿色，辅助使用暖砂色和冷蓝灰

目标：

- 在桌面端和移动端维持一致的软件感
- 避免各页面各自拼接风格不一致的背景和卡片

## 9. 已删除模块说明

以下模块已经从当前代码库删除：

- `detect`
- `result`
- `history`
- `video`
- 与以上模块对应的模型、测试、端点解析和共享组件

删除原则：

- 只保留当前真实后端可支撑的能力
- 不在代码库中长期保留“等以后再接”的运行时代码

## 10. 后续扩展原则

如果未来需要恢复 AI 或视频协作能力，必须同时满足：

1. 先补齐对应后端源码
2. 再补接口契约文档
3. 最后再恢复前端页面、模型和测试

不允许再次通过“先留前端预留模块”的方式制造代码与后端不一致。
