# 斛生 数据字典与枚举规范

## 1. 文档目标

本文档用于统一当前客户端仍在使用的字段、模型、枚举和值域，避免页面、状态层、本地存储和接口层出现语义偏差。

## 2. 通用规则

- JSON 字段统一使用 `camelCase`
- 代码标识符统一使用英文
- 界面文案统一使用中文
- 时间字段按后端真实契约处理
- 可空字段必须明确声明
- `2026-03-11` 的界面重构未引入新的运行时模型和字段
- 当前版本不再维护识别、视频、历史记录相关模型

## 3. 核心模型

### 3.1 `ApiResponse<T>`

用途：

- 前端网络层处理通用包裹协议时使用的泛型模型

字段：

- `code`：业务状态码
- `message`：业务提示
- `data`：业务数据，可空

说明：

- 当前 `web` 分支后端的 7 个已接通接口并不统一使用该结构。
- 该模型保留在网络层作为通用工具，不代表当前后端主契约。

### 3.2 `AppSettings`

用途：

- 本地应用设置

字段：

- `baseUrl`：设备服务基础地址
- `connectTimeoutMs`：连接超时，单位毫秒
- `receiveTimeoutMs`：接收超时，单位毫秒
- `enableLog`：是否开启日志
- `buildFlavor`：构建环境

### 3.3 `AuthUser`

用途：

- 当前登录用户信息

字段：

- `userId`：用户 ID
- `account`：登录账号
- `displayName`：显示名称
- `roles`：角色列表，可为空

### 3.4 `AuthSession`

用途：

- 前端本地登录态对象

字段：

- `accessToken`：前端本地兼容字段
- `refreshToken`：可空，本地替身模式可能使用
- `sessionCookie`：服务端会话 Cookie，可空
- `tokenType`：认证类型，真实后端当前为 `Session`
- `expiresAt`：过期时间，可空
- `loginMode`：登录模式
- `user`：当前用户信息

说明：

- 当前真实后端基于 `HttpSession + JSESSIONID`，不是 Bearer Token。
- `accessToken` 不应被解释为后端真实令牌。

### 3.5 `DeviceStateInfo`

用途：

- `/api/status` 返回的设备状态对象

字段：

- `deviceId`：设备 ID
- `deviceName`：设备名称
- `temperature`：温度值，可空
- `temperatureUnit`：温度单位
- `humidity`：湿度值，可空
- `humidityUnit`：湿度单位
- `light`：光照值，可空
- `lightUnit`：光照单位
- `mq2`：MQ2 值，可空
- `mq2Unit`：MQ2 单位
- `errorCode`：错误码，可空
- `ledOn`：LED 状态，可空
- `updatedAt`：毫秒时间戳

派生语义：

- `alertLevel`：根据错误码映射为 `safe / warning / danger / unknown`
- `alertTitle`：面向界面的状态标题
- `alertDescription`：面向界面的状态说明
- `canControlLed`：是否满足 LED 控制最小条件

### 3.6 `LedOperationReceipt`

用途：

- `/api/ops/led` 的前端接收对象

字段：

- `status`：后端状态值
- `requestId`：请求编号，可空
- `message`：后端消息

### 3.7 `ServiceHealthInfo`

用途：

- 服务健康检查结果

字段：

- `status`：服务状态，例如 `up / down / unknown`
- `responseText`：健康接口原始响应文本
- `checkedAt`：客户端完成检查时间

## 4. 当前枚举

### 4.1 `BuildFlavor`

枚举值：

- `development`
- `staging`
- `production`

### 4.2 `AuthLoginMode`

枚举值：

- `real`
- `mock`

说明：

- `real`：真实后端登录
- `mock`：本地联调替身登录

### 4.3 `PlatformType`

枚举值：

- `android`
- `ios`
- `macos`
- `web`
- `windows`
- `linux`
- `ohos`

### 4.4 `DeviceAlertLevel`

枚举值：

- `safe`
- `warning`
- `danger`
- `unknown`

## 5. 本地存储键

当前仍在使用的本地存储键：

- `app_settings`
- `auth_session`
- `remembered_account`

说明：

- 当前版本已经删除历史记录相关本地结构，不再维护 `history_records`。

## 6. 文档边界

以下内容不再属于当前数据字典维护范围：

- 检测结果模型
- 视频流模型
- 历史记录模型
- 检测类别、严重程度、识别来源等旧枚举

如果未来恢复相应业务，必须在对应代码回归后重新补写本文件，而不是保留过时字段说明。
