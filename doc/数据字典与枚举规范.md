# 斛生 数据字典与枚举规范

## 1. 文档目标

本文档用于统一斛生中的字段定义、枚举值、模型命名和数据边界，避免前后端、页面层、状态层和存储层出现语义不一致。

本文档适用范围：

- Flutter 前端模型
- Java 后端 DTO / VO / Entity
- JSON 接口字段
- 本地历史记录字段
- 测试样例数据

## 2. 基本规则

### 2.1 字段命名规则

- JSON 字段统一使用 `camelCase`
- 代码标识符统一使用英文
- 界面显示文案统一使用中文
- 同一个字段不能在不同层出现不同含义

### 2.2 类型规则

- 时间字段按后端真实契约定义，识别结果使用 ISO 8601 字符串，`/api/status.updatedAt` 使用毫秒时间戳
- 主键 ID 统一使用字符串
- 置信度统一使用 `0 ~ 1` 的 `double`
- 严重程度分级统一使用枚举值，不直接在接口中传中文
- 坐标统一使用归一化比例值

### 2.3 空值规则

- 可空字段必须在文档中明确标注
- 非可空字段缺失时应视为协议错误
- 正式环境不允许通过空对象掩盖字段错误

## 3. 核心实体

### 3.1 ApiResponse

用途：

- 识别相关接口和前端内部模型预留的统一包裹结构

字段定义：

- `code`：`int`，必填，业务状态码
- `message`：`String`，必填，业务提示
- `data`：`dynamic`，可空，业务数据

说明：

- 当前前端代码已落地通用 `ApiResponse<T>` 类型。
- 但本地 Spring Boot 后端的 `/api/login`、`/api/check-login`、`/api/logout`、`/api/status`、`/api/ops/led`、`/api/health` 当前并不统一使用该包裹结构。

### 3.2 DetectRequest

用途：

- 单图识别请求参数描述

字段定义：

- `clientTraceId`：`String`，必填，客户端请求 ID
- `capturedAt`：`String?`，可空，采集时间
- `platform`：`String`，必填，平台类型
- `imageWidth`：`int?`，可空，图片宽度
- `imageHeight`：`int?`，可空，图片高度

### 3.3 DetectResponse

用途：

- 单图识别或实时识别的核心返回结构

字段定义：

- `detectId`：`String`，必填，识别任务 ID
- `sourceType`：`String`，必填，来源类型
- `capturedAt`：`String`，必填，采集时间
- `summary`：`DetectSummary`，必填，结果摘要
- `imageInfo`：`ImageInfo?`，可空，图像信息
- `detections`：`List<DetectionItem>`，必填，可为空列表
- `advice`：`AdviceInfo?`，可空，防治建议
- `modelInfo`：`ModelInfo?`，可空，模型信息

### 3.4 DetectSummary

用途：

- 结果页顶部摘要信息

字段定义：

- `primaryLabelCode`：`String`，必填，主标签编码
- `primaryLabelName`：`String`，必填，主标签名称
- `category`：`String`，必填，识别类别
- `confidence`：`double`，必填，置信度
- `severityLevel`：`String`，必填，严重程度
- `severityScore`：`double?`，可空，严重度分数
- `healthStatus`：`String`，必填，健康状态

### 3.5 DetectionItem

用途：

- 单个检测框信息

字段定义：

- `detectionId`：`String`，必填
- `labelCode`：`String`，必填
- `labelName`：`String`，必填
- `category`：`String`，必填
- `confidence`：`double`，必填
- `severityLevel`：`String`，必填
- `bbox`：`BoundingBox`，必填

### 3.6 BoundingBox

用途：

- 标注框坐标

字段定义：

- `x`：`double`，必填，左上角横向比例值
- `y`：`double`，必填，左上角纵向比例值
- `width`：`double`，必填，宽度比例值
- `height`：`double`，必填，高度比例值

约束：

- 取值范围必须在 `0 ~ 1`
- `x + width` 不应超过 `1`
- `y + height` 不应超过 `1`

### 3.7 AdviceInfo

用途：

- 防治建议信息

字段定义：

- `title`：`String`，必填
- `summary`：`String`，必填
- `preventionSteps`：`List<String>`，必填，可为空列表

### 3.8 ImageInfo

用途：

- 图像原始信息和结果图地址

字段定义：

- `width`：`int`，必填
- `height`：`int`，必填
- `originalUrl`：`String?`，可空
- `annotatedUrl`：`String?`，可空

### 3.9 ModelInfo

用途：

- 识别模型信息

字段定义：

- `modelName`：`String`，必填
- `modelVersion`：`String`，必填
- `inferenceMs`：`int`，必填

### 3.10 HistoryItem

用途：

- 历史记录列表展示项

字段定义：

- `historyId`：`String`，必填
- `detectId`：`String`，必填
- `coverUrl`：`String?`，可空
- `primaryLabelCode`：`String`，必填
- `primaryLabelName`：`String`，必填
- `category`：`String`，必填
- `severityLevel`：`String`，必填
- `confidence`：`double`，必填
- `capturedAt`：`String`，必填

### 3.11 HistoryRecord

用途：

- 历史记录详情模型，组合列表展示信息与完整识别结果

字段定义：

- `item`：`HistoryItem`，必填
- `response`：`DetectResponse`，必填
- `savedAt`：`String`，必填
- `sourceImagePath`：`String?`，可空

### 3.12 AppSettings

用途：

- 前端本地配置

字段定义：

- `baseUrl`：`String`，必填
- `connectTimeoutMs`：`int`，必填
- `receiveTimeoutMs`：`int`，必填
- `enableLog`：`bool`，必填
- `buildFlavor`：`String`，必填

### 3.13 AuthUser

用途：

- 当前登录用户信息

字段定义：

- `userId`：`String`，必填
- `account`：`String`，必填
- `displayName`：`String`，必填
- `roles`：`List<String>`，必填，可为空列表

### 3.14 AuthSession

用途：

- 当前登录会话

字段定义：

- `accessToken`：`String`，必填，当前主要用于前端统一会话模型占位
- `refreshToken`：`String?`，可空，当前本地 Spring Boot 后端未使用
- `sessionCookie`：`String?`，可空，Spring Boot `HttpSession` 对应的 Cookie 值
- `tokenType`：`String`，必填
- `expiresAt`：`String?`，可空
- `loginMode`：`AuthLoginMode`，必填
- `user`：`AuthUser`，必填

### 3.15 ServiceHealthInfo

用途：

- 服务健康检查结果模型

字段定义：

- `status`：`String`，必填，服务状态
- `serviceName`：`String`，必填，服务名称
- `serviceVersion`：`String`，必填，服务版本
- `modelStatus`：`String`，必填，模型状态
- `serverTime`：`String?`，可空，服务时间

### 3.16 DeviceStateInfo

用途：

- `/api/status` 对应的设备状态模型

字段定义：

- `deviceId`：`String`，必填
- `deviceName`：`String`，必填
- `temperature`：`double?`，可空，温度值
- `temperatureUnit`：`String`，必填，默认 `°C`
- `humidity`：`double?`，可空，湿度值
- `humidityUnit`：`String`，必填，默认 `%`
- `light`：`double?`，可空，光照值
- `lightUnit`：`String`，必填，默认 `Lux`
- `mq2`：`double?`，可空，MQ2 值
- `mq2Unit`：`String`，必填，默认 `ppm`
- `errorCode`：`int?`，可空，后端错误码
- `ledOn`：`bool?`，可空，LED 当前状态
- `updatedAt`：`int`，必填，毫秒时间戳

说明：

- 该模型在前端保留了后端 `value + unit` 的单位信息，不再只取数值本身。
- 前端会基于 `errorCode` 进一步映射展示等级和说明文案。

### 3.17 DeviceAlertLevel

用途：

- 前端用于展示设备错误码等级的派生枚举

枚举值：

- `safe`：错误码 `0`
- `warning`：错误码 `1`
- `danger`：错误码 `2`
- `unknown`：空值或未定义错误码
- `serverTime`：`String`，必填，服务端时间

## 4. 枚举规范

### 4.1 BuildFlavor

- `development`：开发环境
- `staging`：测试环境
- `production`：正式环境

### 4.2 SourceType

- `image`：单图识别
- `realtime`：实时识别
- `history`：历史记录详情

### 4.3 DetectionCategory

- `disease`：病害
- `insect`：虫害
- `healthy`：健康
- `unknown`：未知

### 4.4 AuthLoginMode

- `real`：真实接口登录
- `mock`：受控演示登录

### 4.5 SeverityLevel

- `none`：无明显异常
- `low`：轻度
- `medium`：中度
- `high`：重度
- `critical`：严重

### 4.6 HealthStatus

- `healthy`：健康
- `risk`：风险
- `abnormal`：异常
- `unknown`：未知

### 4.7 PlatformType

- `android`
- `ios`
- `macos`
- `web`
- `windows`
- `linux`
- `ohos`

兼容说明：

- `openharmony` 和 `harmonyos` 在当前代码中也会被解析为 `ohos`
- 对外契约和存储值统一使用 `ohos`

### 4.8 DetectTaskStatus

- `idle`：未开始
- `running`：进行中
- `success`：成功
- `failed`：失败

### 4.9 RealtimeSessionStatus

- `idle`：未开始
- `initializing`：初始化中
- `running`：运行中
- `paused`：已暂停
- `permissionDenied`：权限受限
- `error`：异常

## 5. 石斛标签编码规范

标签由两部分组成：

- 大类：`disease` / `insect` / `healthy`
- 具体标签：`black_spot` / `scale_insect` 等

推荐格式：

- `disease_black_spot`
- `disease_anthracnose`
- `disease_soft_rot`
- `disease_stem_rot`
- `insect_scale_insect`
- `insect_aphid`
- `healthy_normal_leaf`
