# 石斛病虫害识别项目 数据字典与枚举规范

## 1. 文档目标

本文档用于统一石斛病虫害识别项目中的字段定义、枚举值、模型命名和数据边界，避免前后端、页面层、状态层和存储层各自定义一套语义。

本文档适用范围：

- Flutter 前端模型
- Java 后端 DTO / VO / Entity
- JSON 接口字段
- 本地历史记录字段
- 测试样例数据

## 2. 基本规则

### 2.1 字段命名规则

- JSON 字段统一使用 `camelCase`
- 代码标识符统一使用英文
- 界面显示文案统一使用中文
- 同一个字段不能在不同层出现不同含义

### 2.2 类型规则

- 时间统一使用 ISO 8601 字符串
- 主键 ID 统一使用字符串
- 置信度统一使用 `0 ~ 1` 的 `double`
- 严重程度分级统一使用枚举值，不直接在接口中传中文
- 坐标统一使用归一化比例值

### 2.3 空值规则

- 可空字段必须在文档中明确标注
- 非可空字段缺失时应视为协议错误
- 正式环境不允许通过空对象掩盖字段错误

## 3. 核心实体

### 3.1 ApiResponse

用途：

- 所有接口统一包裹结构

字段定义：

- `code`：`int`，必填，业务状态码
- `message`：`String`，必填，业务提示
- `data`：`dynamic`，可空，业务数据

### 3.2 DetectRequest

用途：

- 单图识别请求参数描述

字段定义：

- `clientTraceId`：`String`，必填，客户端请求 ID
- `capturedAt`：`String?`，可空，采集时间
- `platform`：`String`，必填，平台类型
- `imageWidth`：`int?`，可空，图片宽度
- `imageHeight`：`int?`，可空，图片高度

### 3.3 DetectResponse

用途：

- 单图识别或实时识别的核心返回结构

字段定义：

- `detectId`：`String`，必填，识别任务 ID
- `sourceType`：`String`，必填，来源类型
- `capturedAt`：`String`，必填，采集时间
- `summary`：`DetectSummary`，必填，结果摘要
- `imageInfo`：`ImageInfo?`，可空，图像信息
- `detections`：`List<DetectionItem>`，必填，可为空列表
- `advice`：`AdviceInfo?`，可空，防治建议
- `modelInfo`：`ModelInfo?`，可空，模型信息

### 3.4 DetectSummary

用途：

- 结果页顶部摘要信息

字段定义：

- `primaryLabelCode`：`String`，必填，主标签编码
- `primaryLabelName`：`String`，必填，主标签名称
- `category`：`String`，必填，识别类别
- `confidence`：`double`，必填，置信度
- `severityLevel`：`String`，必填，严重程度
- `severityScore`：`double?`，可空，严重度分数
- `healthStatus`：`String`，必填，健康状态

### 3.5 DetectionItem

用途：

- 单个检测框信息

字段定义：

- `detectionId`：`String`，必填
- `labelCode`：`String`，必填
- `labelName`：`String`，必填
- `category`：`String`，必填
- `confidence`：`double`，必填
- `severityLevel`：`String`，必填
- `bbox`：`BoundingBox`，必填

### 3.6 BoundingBox

用途：

- 标注框坐标

字段定义：

- `x`：`double`，必填，左上角横向比例值
- `y`：`double`，必填，左上角纵向比例值
- `width`：`double`，必填，宽度比例值
- `height`：`double`，必填，高度比例值

约束：

- 取值范围必须在 `0 ~ 1`
- `x + width` 不应超过 `1`
- `y + height` 不应超过 `1`

### 3.7 AdviceInfo

用途：

- 防治建议信息

字段定义：

- `title`：`String`，必填
- `summary`：`String`，必填
- `preventionSteps`：`List<String>`，必填，可为空列表

### 3.8 ImageInfo

用途：

- 图像原始信息和结果图地址

字段定义：

- `width`：`int`，必填
- `height`：`int`，必填
- `originalUrl`：`String?`，可空
- `annotatedUrl`：`String?`，可空

### 3.9 ModelInfo

用途：

- 识别模型信息

字段定义：

- `modelName`：`String`，必填
- `modelVersion`：`String`，必填
- `inferenceMs`：`int`，必填

### 3.10 HistoryItem

用途：

- 历史记录列表展示项

字段定义：

- `historyId`：`String`，必填
- `detectId`：`String`，必填
- `coverUrl`：`String?`，可空
- `primaryLabelCode`：`String`，必填
- `primaryLabelName`：`String`，必填
- `category`：`String`，必填
- `severityLevel`：`String`，必填
- `confidence`：`double`，必填
- `capturedAt`：`String`，必填

### 3.11 AppSettings

用途：

- 前端本地配置

字段定义：

- `baseUrl`：`String`，必填
- `connectTimeoutMs`：`int`，必填
- `receiveTimeoutMs`：`int`，必填
- `enableLog`：`bool`，必填
- `buildFlavor`：`String`，必填

## 4. 枚举规范

### 4.1 SourceType

- `image`：单图识别
- `realtime`：实时识别
- `history`：历史记录详情

### 4.2 DetectionCategory

- `disease`：病害
- `insect`：虫害
- `healthy`：健康
- `unknown`：未知

### 4.3 SeverityLevel

- `none`：无明显异常
- `low`：轻度
- `medium`：中度
- `high`：重度
- `critical`：严重

### 4.4 HealthStatus

- `healthy`：健康
- `risk`：风险
- `abnormal`：异常
- `unknown`：未知

### 4.5 PlatformType

- `android`
- `ios`
- `web`
- `windows`
- `linux`

### 4.6 DetectTaskStatus

- `idle`：未开始
- `running`：进行中
- `success`：成功
- `failed`：失败

## 5. 石斛标签编码规范

标签由两部分组成：

- 大类：`disease` / `insect` / `healthy`
- 具体标签：`black_spot` / `scale_insect` 等

推荐格式：

- `disease_black_spot`
- `disease_anthracnose`
- `disease_soft_rot`
- `disease_stem_rot`
- `disease_yellow_leaf`
- `insect_scale_insect`
- `insect_red_spider`
- `insect_aphid`
- `healthy_normal`

## 6. 当前建议的石斛识别标签表

### 6.1 病害标签

- `disease_black_spot`：黑斑病
- `disease_anthracnose`：炭疽病
- `disease_soft_rot`：软腐病
- `disease_stem_rot`：茎腐病
- `disease_yellow_leaf`：黄叶 / 黄化异常

### 6.2 虫害标签

- `insect_scale_insect`：介壳虫
- `insect_red_spider`：红蜘蛛
- `insect_aphid`：蚜虫

### 6.3 健康标签

- `healthy_normal`：健康植株

说明：

- 如果模型后续扩展新类别，必须同步更新本节和接口契约文档
- 前后端标签编码必须完全一致，不允许前端自行翻译编码

## 7. 显示文案规范

- 代码中使用 `labelCode`
- UI 显示使用 `labelName`
- 严重程度展示统一走 `severityLevel` 到中文映射
- 健康状态展示统一走 `healthStatus` 到中文映射

映射示例：

- `low` -> `轻度`
- `medium` -> `中度`
- `high` -> `重度`
- `critical` -> `严重`

## 8. 前端落地要求

- 所有模型字段与本文档保持一致
- 新增字段前必须先更新文档
- 删除字段前必须确认兼容影响
- 枚举字段不得直接写死中文判断逻辑
- JSON 序列化类、状态管理类、页面展示逻辑都必须引用同一套模型定义
