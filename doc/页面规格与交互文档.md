# 斛生 页面规格与交互文档

## 1. 文档目标

本文档用于定义当前客户端各页面的模块结构、交互行为、状态切换和跨平台表现。本文档只记录当前代码已经实现的页面，不再记录已删除或未接入后端的预留页面。

## 2. 全局规则

### 2.1 路由集合

当前运行时只保留以下路由：

- `/`
- `/login`
- `/home`
- `/realtime-detect`
- `/settings`
- `/about`

路由规则：

- `SplashPage` 负责启动分流
- 匿名用户访问受保护页面时跳转到 `LoginPage`
- 已登录用户访问 `LoginPage` 时跳转到 `RealtimeDetectPage`

### 2.2 全局状态规则

核心页面至少覆盖以下状态：

- `loading`
- `success`
- `error`

补充说明：

- 不再维护识别、视频、历史记录相关页面状态机

### 2.3 全局反馈规则

- 异步动作必须有加载态或禁用态
- 错误必须可见，不允许静默失败
- 危险操作必须二次确认
- Snackbar 仅用于即时反馈，不替代页面级错误状态

### 2.4 跨平台与视觉规则

- 手机端优先纵向单列布局
- 桌面端优先侧边导航与更宽卡片布局
- 首页、主控台、设置页、系统总览页统一接入工作台壳层
- 紧凑宽度下统一切换为底部导航
- 主视觉统一使用青绿色主色，辅助使用暖砂色和冷蓝灰
- 桌面端支持 hover 与焦点反馈
- 页面首屏区块统一使用轻量揭示动效，保持进入节奏一致
- 主题、背景、卡片圆角和阴影保持统一的 Material 3 风格

## 3. 页面规格

### 3.1 启动页 `SplashPage`

页面目标：

- 完成冷启动初始化
- 加载本地设置、应用版本和登录态
- 跳转主入口

页面模块：

- Logo 区
- 项目名称区
- 状态提示区
- 重试按钮区

状态：

- `loading`
- `error`

交互规则：

- 初始化成功后进入 `LoginPage` 或 `RealtimeDetectPage`
- 初始化失败时必须提供明确错误文案和重试入口

### 3.2 登录页 `LoginPage`

页面目标：

- 承接登录和注册
- 展示认证方式与服务地址

页面模块：

- 返回系统概览入口
- 左侧说明面板
- 登录 / 注册切换区
- `账号 / 密码 / 确认密码` 表单区
- 记住账号入口
- 演示环境提示区，仅在演示模式下显示

状态：

- `idle`
- `submitting`
- `error`
- `success_notice`

交互规则：

- 登录和注册提交中必须禁用重复点击
- 注册成功后回到登录模式展示成功提示
- 说明区必须明确展示 `HttpSession + JSESSIONID`
- 服务地址可在登录页恢复为默认值
- 返回系统总览使用顶层路由替换，不叠加登录页返回栈
- 窄宽度下自动变为单栏布局

### 3.3 工作台首页 `HomePage`

页面目标：

- 展示当前值守基线
- 提供统一业务入口

页面模块：

- 工作台页头
- 首页总览区
- 功能入口区
- 当前值守范围区
- 实时快照区

当前入口：

- 监控主控台
- 系统概览
- 运维设置

交互规则：

- 入口卡片必须尽量在首屏 Hero 后直接可见
- 设备摘要失败时必须展示重试入口
- 首页自动每 8 秒刷新一次设备快照和服务巡检
- 桌面端入口卡片支持 hover 位移与缩放反馈
- 桌面端一级导航置于页头导航带，移动端收为底部导航
- 窄宽度下入口卡片自动收为单列

### 3.4 实时监控页 `RealtimeDetectPage`

页面目标：

- 承接设备状态轮询和 LED 控制
- 作为登录后的核心值守页面

页面模块：

- 工作台页头
- 顶部值守台
- Hero 主状态区
- 四项环境指标区
- 运行明细与 LED 控制区
- 状态说明区

状态：

- `loading`
- `success`
- `error`

交互规则：

- 首次无状态时仍需展示稳定页面骨架
- 自动刷新可切换，默认按 3 秒轮询
- 手动刷新可随时触发
- LED 控制提交中必须禁用重复操作
- LED 控制成功后需要补一次延迟回刷，等待设备状态回写
- 退出登录前必须二次确认
- 桌面端通过页头导航带切换模块，移动端通过底部导航切换模块
- 刷新失败时保留最近一次成功状态，并提示错误信息

### 3.5 运维设置页 `SettingsPage`

页面目标：

- 集中处理服务配置、健康检查和本机数据

页面模块：

- 工作台页头
- 运行环境卡片
- 服务配置卡片
- 服务健康卡片
- 设备状态卡片
- 登录会话卡片
- 本机数据卡片
- 系统总览入口卡片

当前主操作：

- 查看当前平台、版本、环境
- 查看设备服务地址与超时配置
- 修改服务地址，仅允许当前构建开放的高风险配置入口
- 查看 `/api/health` 原始响应
- 查看设备状态并执行 LED 控制
- 清除记住账号
- 恢复默认设置
- 退出登录

交互规则：

- 正式环境隐藏高风险编辑入口
- 清除记住账号、恢复默认设置与退出登录必须二次确认
- 恢复默认设置后立即持久化
- 设置页自动每 12 秒刷新一次健康检查和设备状态
- 桌面端通过页头导航带切换模块，移动端通过底部导航切换模块
- 设置页卡片纵向排列，窄宽度不出现横向滚动

### 3.6 系统总览页 `AboutPage`

页面目标：

- 解释平台价值、业务闭环和栽培背景

页面模块：

- 工作台页头
- Hero 区
- 业务架构区
- 栽培背景区
- 调控目标区

交互规则：

- 已登录用户可从页头直接返回首页或主控台
- 未登录用户可从页头直接进入登录页
- 桌面端通过页头导航带切换模块，移动端通过底部导航切换模块
- 说明内容在桌面端使用双栏组织，在窄宽度下自动折为单列

## 4. 已删除页面说明

以下页面不再属于当前代码库，也不再属于页面规格范围：

- `DetectPage`
- `ResultPage`
- `HistoryPage`
- `VideoHubPage`
- `VideoStreamDetailPage`
