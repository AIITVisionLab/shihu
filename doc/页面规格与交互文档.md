# 斛生 页面规格与交互文档

## 1. 文档目标

本文档用于定义当前客户端各页面的模块结构、交互行为、状态切换和跨平台表现。本文档只记录当前代码已经实现的页面，不再记录已删除或未接入后端的预留页面。

## 2. 全局规则

### 2.1 路由集合

当前运行时只保留以下路由：

- `/`
- `/login`
- `/home`
- `/realtime-detect`
- `/video`
- `/settings`
- `/about`

路由规则：

- `SplashPage` 负责启动分流
- 匿名用户访问受保护页面时跳转到 `LoginPage`
- 已登录用户访问 `LoginPage` 时跳转到 `RealtimeDetectPage`

### 2.2 全局状态规则

核心页面至少覆盖以下状态：

- `loading`
- `success`
- `error`

补充说明：

- 不再维护识别、历史记录相关页面状态机

### 2.3 全局反馈规则

- 异步动作必须有加载态或禁用态
- 错误必须可见，不允许静默失败
- 危险操作必须二次确认
- Snackbar 仅用于即时反馈，不替代页面级错误状态

### 2.4 跨平台与视觉规则

- 手机端优先纵向单列布局
- 桌面端优先侧边导航与更宽卡片布局
- 首页、主控台、设置页、系统总览页统一接入工作台壳层
- 紧凑宽度下统一切换为底部导航
- 主视觉统一使用透明兰花品牌图标，搭配青松绿、雾青、亚麻和淡紫灰
- 主视觉统一使用 `#518463` 青松绿主色，辅以 `#A7D3B2 / #CBF2E0 / #D2C8AC / #CEBBD8`
- 背景底色统一收为偏冷雾白纸感，不再回到偏黄暖灰底
- 桌面端支持 hover 与焦点反馈
- 一级导航切换不重复播放首屏入场动效，已访问板块保持原状态
- 主题、背景、卡片圆角和阴影统一收敛为浅纸感工作台风格
- 工作台壳层统一使用悬浮导航仓、标题控制仓和背景网格氛围
- 桌面端卡片编排最多保持两列，避免首屏出现过多碎片卡片
- 手机端优先单列阅读顺序，指标类信息允许使用双项网格
- 首页、值守、视频、设置和帮助页统一使用随屏宽回落的内容区留白，避免手机端出现过宽内边距
- 桌面端主内容区统一限宽在 `1360` 内，窗口全屏后不继续横向拉伸主卡，同时减少放大窗口后的无效留白
- 首页、值守、视频、设置和帮助页的首卡统一使用“主信息区 + 固定侧区”的比例结构，不做各页不同的大 Hero 头重布局
- 首页、值守、视频、设置和帮助页的首卡侧区统一按 `328-392` 自适应宽度收口，不再各页单独写固定 `340`
- 首页、值守、设置等主体双栏统一使用相同断点、间距和均衡列宽，避免左右比例在不同页面反复变化
- 首页、值守、视频、设置和帮助页的主体大卡统一保持白纸底，只在标题线、小标签和局部光晕里保留浅色强调
- 全站二级卡片、导航标签、登录辅助区和视频入口统一复用首页“环境速览”的浅松绿、雾青、亚麻、淡紫强调顺序
- 二级信息卡统一以白底为主，使用左上短色线和清晰但克制的浅色染面表达层级，不再依赖整块重色底
- 二级信息块统一使用同一套“标签 + 值 + 必要说明”的卡片语法，不重复叠加分隔线、双色盒和独立装饰条
- 重复说明、重复状态和跨页重复操作默认不展示
- 用户可见文案不暴露“工作台”“主路径”“轨道”等内部设计术语
- 通用卡片统一使用高光顶线和一致标题层级，不再各页单独设计卡片标题样式
- Hero 与导航仓只保留弱化线条、光晕和渐变层，不使用独立的配色说明模块
- 工作台页头统一使用与正文同宽的紧凑标题条，不再做独立大卡
- 登录页桌面端使用说明区 + 表单区分栏布局，手机端收为单列一体化结构
- 登录页的体验入口和服务切换提示只作为辅助信息展示，不直出原始服务地址
- 值守、视频、设置、帮助页统一使用“Hero 总览 + 两列主体 / 单列回落”的编排方式

## 3. 页面规格

### 3.1 启动页 `SplashPage`

页面目标：

- 完成冷启动初始化
- 加载本地设置、应用版本和登录态
- 跳转主入口

页面模块：

- 启动图标区
- 启动状态卡片区
- 失败重试区

状态：

- `loading`
- `error`

交互规则：

- 初始化成功后进入 `LoginPage` 或 `RealtimeDetectPage`
- 初始化失败时必须提供明确错误文案和重试入口

### 3.2 登录页 `LoginPage`

页面目标：

- 承接登录和注册
- 展示登录主表单，必要时补充异常提示

页面模块：

- 返回系统概览入口
- 浅纸感工作台式认证壳层
- 登录说明区
- 登录主表单区
- 服务切换辅助提示卡，仅在检测到自定义配置时显示
- 登录 / 注册切换区
- `账号 / 密码 / 确认密码` 表单区
- 记住账号入口

状态：

- `idle`
- `submitting`
- `error`
- `success_notice`

交互规则：

- 登录和注册提交中必须禁用重复点击
- 注册成功后回到登录模式展示成功提示
- 登录页展示“直接预览界面”入口，写入本地预览会话后进入工作台，用于后端暂不可用时查看整体编排
- 仅在检测到自定义服务配置时展示轻量提示和恢复默认入口，不直出原始服务地址
- 返回系统总览使用顶层路由替换，不叠加登录页返回栈
- 桌面端优先使用“登录说明 + 独立表单卡片”双区布局，移动端收为单列纵向结构
- 登录说明区聚焦“默认进入值守台”和总览 / 视频 / 我的的职责划分，不重复首页口号式导语

### 3.3 工作台首页 `HomePage`

页面目标：

- 展示当前值守基线
- 承接设备状态总览

页面模块：

- 工作台页头
- 总览 Hero 区
- 环境速览区

交互规则：

- 设备摘要失败时必须展示重试入口
- 首页自动每 8 秒刷新一次设备快照
- 首页页面层不再持有轮询 Timer，自动刷新由应用层 Provider 管理
- 首页 Hero 允许使用信号盘面、轨道编号和指标矩阵这类更强的视觉组织方式
- 桌面端一级导航置于页头导航带，移动端收为底部导航
- 桌面端品牌与当前账号统一收在左侧导航仓，页头不再重复展示账号标签
- 首页正文不再重复展示一级导航入口卡
- 首页信息顺序固定为总览、快照
- 首页默认不展示 `deviceId` 等技术兜底信息
- 首页桌面端优先扩展正文宽度，不再维持偏窄画布造成大面积空白

### 3.4 实时监控页 `RealtimeDetectPage`

页面目标：

- 承接设备状态轮询和 LED 控制
- 作为登录后的核心值守页面

页面模块：

- 工作台页头
- Hero 主状态区
- Hero 内嵌值守节奏区
- 四项环境指标区
- 设备、LED 与处理建议整合区
- AI 巡检结果区
- 桌面端保持“指标主列 + 操作说明侧列”，手机端回落为单列

状态：

- `loading`
- `success`
- `error`

交互规则：

- 首次无状态时仍需展示稳定页面骨架
- 自动刷新可切换，默认按 3 秒轮询
- 手动刷新可随时触发
- LED 控制提交中必须禁用重复操作
- LED 控制成功后需要补一次延迟回刷，等待设备状态回写
- 监控启动、轮询停止和 LED 延迟回刷由 Controller 托管，页面层只触发事件
- 桌面端通过页头导航带切换模块，移动端通过底部导航切换模块
- 刷新失败时保留最近一次成功状态，并提示错误信息
- 主控台信息顺序固定为状态、节奏、指标、操作整合区
- 值守页默认不直出 `deviceId`、错误码等技术字段
- 桌面端优先使用“指标主列 + 操作说明侧列”的布局，减少同级碎片卡片
- 四项环境指标统一收为矩阵式卡片，手机端优先双项网格
- 设备摘要、补光控制与处理建议统一收在同一张操作卡片里
- 值守页优先删除重复说明型小板块，只保留结论、指标和补光控制三类核心信息
- 值守页桌面端指标区与操作区统一按同一套双栏规则排布，不再单独使用不一致的列宽比例
- 值守页桌面端下半区保持紧凑双列，不通过固定高度或机械等高制造空白
- 值守页右侧操作卡优先使用紧凑横向色块，避免放大窗口后说明文案拖成长列
- 值守页 Hero、指标卡和控制卡复用总览页的浅松绿、雾青、亚麻、淡紫配色，不单独发明一套偏灰色面
- AI 巡检区默认展示“最新结论 + 最近记录”，没有结果时展示稳定空状态
- AI 巡检区只消费后端汇总结果，不回退到本地图像叠框或旧识别页

### 3.5 运维设置页 `SettingsPage`

页面目标：

- 集中处理当前设备、登录账号和本机偏好

页面模块：

- 工作台页头
- 当前使用总览卡片
- 账号与本机主卡
- 平台日志卡片
- 手机端与桌面端统一保持单列纵向阅读顺序

当前主操作：

- 查看当前设备、当前账号、最近同步、数据状态和补光状态
- 查看最近平台事件
- 按关键字、事件类型和条数范围筛选平台日志
- 清除记住账号
- 恢复默认设置
- 退出登录

交互规则：

- 默认不展示服务地址等高风险配置项
- 清除记住账号、恢复默认设置与退出登录必须二次确认
- 恢复默认设置后立即持久化
- 使用帮助入口并入账号与本机主卡，不再单独占一张卡
- 当前平台若不支持安全持久化登录态，账号与本机主卡必须明确提示“关闭应用后需要重新登录”
- 桌面端通过页头导航带切换模块，移动端通过底部导航切换模块
- 设置页窄宽度下统一改为单列，不出现横向滚动
- 当前使用总览采用 Hero 式摘要区 + 信息矩阵，不再退回普通表单列表
- 设置页账号与本机主卡减少重复说明和重复装饰，避免页面继续碎片化
- 账号与本机主卡内部统一收敛为“主信息区 + 快捷操作区 + 必要提醒条”，不再拆成独立会话卡和本地偏好卡
- 平台日志卡默认展示最近 6 条事件，支持按关键字、事件类型和条数范围重新查询，并继续优先展示摘要与类型

### 3.6 视频中心页 `VideoPage`

页面目标：

- 展示当前可查看的实时画面
- 让用户在软件内直接查看画面或备用入口

页面模块：

- 工作台页头
- 当前主画面直播放卡
- 其它画面紧凑入口区
- 手机端统一回到单列，桌面端保持紧凑入口布局

交互规则：

- 页面调用 `GET /api/video/streams`
- 当业务后端尚未提供 `/api/video/streams` 时，按当前运行配置回退到单路公网视频网关入口
- 首个可直接查看的在线画面默认提升为“当前主画面”，并在页面内直接开始播放
- 其它画面入口统一使用紧凑切换列表，不再重复展开为多张同级大卡，也不再额外套一层外部卡片
- 视频流卡片允许使用伪预览舞台、监看屏边框和状态灯等视觉元素，但不伪造真实播放器
- 主画面卡只保留“放大查看”和“备用入口”
- 单路画面时不重复展示第二张同内容卡片
- 视频页桌面端主卡侧区跟随统一宽度令牌自适应收口，不再单独固定控制栏宽度
- 视频流入口需要展示 `aiResultForwarded` 状态，提示当前流是否已经联动到后端 AI 汇总链路
- Android / iOS / macOS 点击入口时优先进入软件内播放页
- Windows / Linux 点击入口时优先进入软件内播放页，由原生播放器直接拉取 go2rtc 媒体流
- Web 点击入口时优先进入应用内播放页，由浏览器原生 `iframe` 直接承接跨源播放地址
- 当前暂不支持软件内播放的平台必须明确提示平台限制
- 空状态不展示播放器壳子，失败态不伪造成功内容
- 页面默认不展示地址、主机、端口等技术字段

### 3.7 使用说明页 `AboutPage`

页面目标：

- 解释页面怎么使用和查看重点

页面模块：

- 工作台页头
- Hero 区
- 主路径与信息合并导览区
- 页面整体保持导览式结构，不再退回说明文档式长段落

交互规则：

- 已登录用户可从页头直接返回首页或主控台
- 已登录用户可从页头直接返回设置页
- 未登录用户可从页头直接进入登录页
- 未登录用户可从页头直接打开界面预览，预览模式只使用本地样例数据
- 主路径说明必须按统一轨道展示，不把同类信息拆成碎片卡片
- 页面重点信息必须并入同一张导览卡，不再追加第二张重复说明卡
- 帮助页 Hero 允许使用导览面板、顺序图和编号卡片强化“先看什么、后做什么”
- 说明内容必须全部使用用户视角，不暴露后端接入、接口协作等技术过程
- 桌面端信息说明使用双栏组织，在窄宽度下自动折为单列

## 4. 已删除页面说明

以下页面不再属于当前代码库，也不再属于页面规格范围：

- `DetectPage`
- `ResultPage`
- `HistoryPage`
- `VideoStreamDetailPage`

如未来恢复这些页面，必须在代码回归后重新补写本文件，而不是复用旧描述。
