# 石斛病虫害识别项目 页面规格与交互文档

## 1. 文档目标

本文档用于定义石斛病虫害识别项目 Flutter 前端各页面的模块结构、交互行为、状态切换、异常处理和跨平台表现，作为页面实现与联调验收的直接依据。

## 2. 全局交互规则

### 2.1 路由规则

主流程固定为：

- `SplashPage -> HomePage`
- `HomePage -> DetectPage`
- `HomePage -> RealtimeDetectPage`
- `HomePage -> HistoryPage`
- `HomePage -> SettingsPage`
- `DetectPage -> ResultPage`
- `HistoryPage -> ResultPage`
- `SettingsPage -> AboutPage`

### 2.2 全局状态规则

所有核心页面至少覆盖以下状态：

- `loading`
- `success`
- `empty`
- `error`
- `permissionDenied`，如涉及摄像头或文件权限

### 2.3 全局反馈规则

- 点击后 100ms 内给出交互反馈
- 异步请求必须进入加载态
- 错误必须进入明确失败态，不允许静默降级
- 成功操作需有轻量成功反馈
- 危险操作必须二次确认

### 2.4 动画规则

- 页面切换统一使用轻量转场
- 卡片与信息块允许淡入或位移动画
- 实时识别页优先保证稳定，不叠加复杂特效
- 动画不能影响读图、点击和滚动

### 2.5 跨平台适配规则

- 手机端优先纵向布局
- Web、Windows、Linux 优先居中内容区和更宽的卡片布局
- 桌面端补充 hover、focus、键盘快捷操作
- 各平台功能入口顺序和主流程保持一致

## 3. 页面规格

### 3.1 启动页 `SplashPage`

#### 页面目标

- 完成应用冷启动
- 初始化环境配置、日志、基础本地状态
- 跳转主入口

#### 页面模块

- Logo 区
- 项目名称区
- 简短状态提示区

#### 页面状态

- `loading`：初始化中
- `error`：初始化失败

#### 交互规则

- 默认停留 1 到 2 秒
- 初始化成功自动进入首页
- 初始化失败显示错误提示和重试按钮

#### 完成标准

- 不白屏
- 不长时间卡住
- 初始化失败可恢复

### 3.2 首页 `HomePage`

#### 页面目标

- 作为主导航入口
- 清晰表达项目定位为石斛病虫害识别

#### 页面模块

- 项目标题区
- 石斛业务简介区
- 功能入口区
- 版本与状态信息区

#### 功能入口

- 开始识别
- 实时监测
- 历史记录
- 设置

#### 交互规则

- 功能入口必须一屏可见
- 支持键盘焦点切换和回车触发
- 首页视觉完成度必须高于其他普通页面

#### 异常处理

- 首页本身不依赖接口才能展示
- 如健康检查失败，仅显示状态提示，不阻塞导航

### 3.3 单图识别页 `DetectPage`

#### 页面目标

- 选择或拍摄石斛图片
- 发起单图识别
- 跳转结果页

#### 页面模块

- 图片选择区
- 图片预览区
- 识别参数区，可预留
- 主操作按钮区
- 状态提示区

#### 主操作

- 从相册选择
- 拍照
- 开始识别
- 重新选择

#### 页面状态

- `idle`：未选择图片
- `selected`：已选择图片
- `running`：识别中
- `error`：识别失败

#### 交互规则

- 未选择图片时禁用开始识别按钮
- 识别中禁用重复提交
- 识别成功后自动跳转结果页

#### 异常处理

- 文件读取失败：提示重新选择
- 上传失败：提示重试
- 返回结构异常：进入错误态，不允许假结果兜底

#### 桌面端要求

- 支持拖拽文件到预览区域
- 支持文件选择快捷入口

### 3.4 实时识别页 `RealtimeDetectPage`

#### 页面目标

- 打开摄像头预览
- 展示实时检测框和识别摘要

#### 页面模块

- 预览画面层
- Overlay 标注层
- 顶部状态栏
- 底部结果摘要卡片
- 操作按钮区

#### 主操作

- 开始实时识别
- 暂停
- 返回

#### 页面状态

- `initializing`
- `running`
- `paused`
- `permissionDenied`
- `error`

#### 交互规则

- 初次进入先请求权限
- 权限通过后初始化摄像头
- 识别运行中平滑刷新 overlay
- 暂停后停止帧请求或停止结果更新

#### 异常处理

- 权限拒绝：显示授权说明和返回入口
- 摄像头初始化失败：显示失败提示
- 当前平台不支持：显示明确说明，不允许假预览

#### 桌面端要求

- 支持窗口缩放后画面与标注框同步调整
- 支持空格键暂停 / 恢复，可选

### 3.5 结果页 `ResultPage`

#### 页面目标

- 展示一次识别任务的完整结果
- 展示石斛病害 / 虫害类型、严重程度和防治建议

#### 页面模块

- 结果图区域
- 摘要信息区
- 检测列表区
- 严重程度区
- 防治建议区
- 保存记录按钮区

#### 页面状态

- `success`
- `empty`
- `error`

#### 交互规则

- 页面进入后优先展示摘要
- 详细信息可分块渐进出现
- 保存记录后给出成功反馈

#### 异常处理

- `detections` 为空但接口成功：展示“未检测到明显异常”
- 结果结构不完整：进入错误态并记录日志
- 保存失败：提示重试，不伪装成功

### 3.6 历史记录页 `HistoryPage`

#### 页面目标

- 展示历史识别记录
- 支持检索、查看详情、删除

#### 页面模块

- 筛选区，可预留
- 历史列表区
- 空状态区
- 删除与清空操作区

#### 主操作

- 查看详情
- 删除单条
- 清空全部

#### 页面状态

- `loading`
- `success`
- `empty`
- `error`

#### 交互规则

- 默认按时间倒序
- 点击记录进入结果详情页
- 删除和清空必须二次确认

#### 异常处理

- 列表加载失败：显示重试
- 详情记录不存在：提示记录已失效

### 3.7 设置页 `SettingsPage`

#### 页面目标

- 管理本地配置和运行环境信息

#### 页面模块

- 服务配置区
- 平台与版本信息区
- 缓存管理区
- 历史记录管理区
- 关于项目入口

#### 主操作

- 查看版本
- 查看平台
- 修改 `baseUrl`，仅开发 / 测试环境
- 清除缓存
- 清除历史记录

#### 页面状态

- `success`
- `error`

#### 交互规则

- 正式环境默认隐藏高风险配置项
- 修改配置后立即持久化
- 清空动作必须二次确认

#### 异常处理

- 保存配置失败：提示失败并保留原值
- 清空失败：提示失败，不静默吞错

### 3.8 关于页 `AboutPage`

#### 页面目标

- 展示项目背景、技术栈、版本信息

#### 页面模块

- 项目简介区
- 技术栈区
- 版本信息区
- 团队或支持信息区

## 4. 通用组件规格

### 4.1 CommonButton

要求：

- 主次按钮视觉区分明确
- 支持 `normal`、`loading`、`disabled`
- 支持桌面端 hover 态

### 4.2 CommonCard

要求：

- 统一圆角、阴影、边距
- 统一标题与内容间距
- 支持桌面端宽布局

### 4.3 LoadingView

要求：

- 可用于全页加载和局部加载
- 支持文本提示
- 不使用夸张长动画

### 4.4 EmptyView

要求：

- 必须包含状态说明
- 必须包含下一步建议

### 4.5 ResultInfoCard

要求：

- 支持类别、置信度、严重程度、防治建议
- 支持颜色标签和图标

## 5. 状态与异常矩阵

以下场景必须有页面表现：

- 无网络
- 服务不可用
- 图片无效
- 权限被拒绝
- 结果为空
- 历史记录为空
- 配置保存失败

要求：

- 每种异常都要有明确提示文案
- 能重试的必须给重试入口
- 不能恢复的必须给退出或返回入口

## 6. 完成标准

页面规格完成后，至少满足：

- 每个页面模块结构明确
- 每个页面的主按钮和主流程明确
- 每个页面状态明确
- 每个页面失败态明确
- 多平台交互差异明确
- 可直接据此拆组件、写状态和接接口

## 7. 结论

本文档用于保证石斛病虫害识别项目的页面实现一致、交互一致、异常处理一致。后续所有页面开发、组件拆分、状态管理和联调行为，均以本文档为准。
