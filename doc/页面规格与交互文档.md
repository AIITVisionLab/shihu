# 石斛幼苗智能培育管理平台 页面规格与交互文档

## 1. 文档目标

本文档用于定义石斛幼苗智能培育管理平台 Flutter 前端各页面的模块结构、交互行为、状态切换、异常处理和跨平台表现，作为页面实现和文档验收的直接依据。

说明：

- 本文档只记录当前代码已经实现或已经明确预留的页面行为。
- 对尚未接入的能力，会明确标注为“占位”或“后续接入”，不把规划项写成已实现项。

## 2. 全局交互规则

### 2.1 路由规则

当前主流程固定为：

- `SplashPage -> LoginPage / RealtimeDetectPage`
- `LoginPage -> RealtimeDetectPage`
- `RealtimeDetectPage -> HomePage / SettingsPage / 退出登录`
- `HomePage -> AboutPage`
- `HomePage -> RealtimeDetectPage`
- `HomePage -> SettingsPage`
- `DetectPage -> HomePage / RealtimeDetectPage`
- `HistoryPage -> ResultPage`
- `SettingsPage -> AboutPage`

### 2.2 全局状态规则

当前核心页面至少覆盖以下状态：

- `loading`
- `success`
- `empty`
- `error`

补充说明：

- 摄像头权限状态当前不在页面主链路中，待独立识别服务并入后再恢复对应页面状态。

### 2.3 全局反馈规则

- 点击后应立即给出交互反馈
- 异步动作必须进入加载态或禁用重复提交
- 错误必须展示为明确失败态或错误提示
- 删除、清空等危险操作必须二次确认
- 不允许用假成功值掩盖真实失败

### 2.4 跨平台适配规则

- 手机端优先纵向布局
- Web、Windows、Linux、macOS 优先居中内容区和更宽的卡片布局
- 桌面端补充 hover、focus 和键盘可用性
- Android、iOS、Web、Windows、Linux、macOS、OpenHarmony / 鸿蒙 的入口顺序和主流程保持一致
- 平台差异主要收敛到文件选择、权限、窗口尺寸和运行时判断

## 3. 页面规格

### 3.1 启动页 `SplashPage`

页面目标：

- 完成冷启动初始化
- 加载本地设置、应用版本信息和登录态
- 跳转主入口

页面模块：

- Logo 区
- 项目名称区
- 标语区
- 状态提示区

当前页面状态：

- `loading`：初始化中
- `error`：初始化失败

交互规则：

- 初始化成功后按登录态进入首页或登录页
- 初始化失败时显示错误信息和“重试初始化”按钮
- 启动页不允许白屏

### 3.2 首页 `HomePage`

页面目标：

- 作为主导航入口
- 展示平台定位和当前后端能力

页面模块：

- 标题、版本和当前用户区
- 基线说明区
- 后端能力说明区
- 设备摘要区
- 功能入口卡片区

当前入口：

- 监控主控台
- 公开预览
- 运维设置
- 单图识别扩展位
- 识别历史扩展位

交互规则：

- 入口按钮一屏可见
- 桌面端支持焦点和点击反馈
- 首页文案必须明确当前已实现和待补能力
- 后端未提供的能力只能以下一阶段扩展位展示，不能伪装成可用主功能入口
- 首页设备摘要失败时必须给出重试入口，不允许静默空白

### 3.3 单图识别页 `DetectPage`

页面目标：

- 明确告知当前后端未接入独立识别服务
- 保留后续独立识别服务并入后的页面扩展位

页面模块：

- 未接入说明卡片
- 返回平台总览入口
- 前往监控主控台入口

当前页面状态：

- `unavailable`

交互规则：

- 当前后端不具备识别能力时，页面必须展示未接入说明而不是继续走 mock 成功链路
- 页面应给出返回总览和回到监控主控台的明确操作
- 识别服务并入后，可恢复现有识别控制器、结果页和历史记录链路

当前实现边界：

- 桌面端拖拽入口仅在文案中预留，当前尚未实现
- 当前 `web` 分支后端不包含 `/api/v1/detect/image`，因此页面默认展示未接入态
- 如需恢复识别主链路，必须先提供独立识别服务并显式打开对应前端能力开关

### 3.4 实时监控页 `RealtimeDetectPage`

页面目标：

- 对齐后端设备监控主控台
- 承接实时状态轮询、错误码映射和 LED 控制

当前实现状态：

- 当前页面已接入顶部链路状态区、设备主横幅、四项环境指标区、运行明细区、LED 控制区和错误码说明区
- 页面进入后自动启动 `/api/status` 轮询
- 页面支持手动刷新和自动刷新开关
- LED 控制提交成功后会继续刷新等待状态回写

当前页面状态：

- `loading`
- `success`
- `error`

当前交互规则：

- 用户进入页面后必须能看到稳定的监控结构，不允许白屏
- 自动刷新关闭后不再继续轮询
- 刷新失败时保留最近一次成功状态，并展示明确错误信息
- LED 控制中必须禁用重复提交
- 当 `/api/status` 尚未返回非空 `deviceId` 时，LED 控制入口必须禁用，并明确提示等待设备身份上报

### 3.5 结果页 `ResultPage`

页面目标：

- 展示一次识别任务的完整结果
- 支持保存到历史记录

页面模块：

- 结果图区域
- 摘要信息区
- 检测列表区
- 防治建议区
- 保存按钮区

当前页面状态：

- `success`
- `empty`
- `error`

交互规则：

- 优先展示摘要结果
- 当 `detections` 为空时展示“未检测到明显异常”
- 从单图识别进入时允许保存
- 从历史记录进入时禁用再次保存

图片展示规则：

- 优先使用当前识别源图片路径
- 其次使用 `annotatedUrl`
- 再次回退到 `originalUrl`

### 3.6 历史记录页 `HistoryPage`

页面目标：

- 展示本地历史记录
- 支持查看详情、删除单条、清空全部

页面模块：

- 顶部操作区
- 历史列表区
- 空状态区

当前页面状态：

- `loading`
- `success`
- `empty`
- `error`

交互规则：

- 点击列表项进入结果页
- 删除单条前二次确认
- 清空全部前二次确认
- 记录为空时展示空状态提示

缩略图规则：

- 优先展示 `sourceImagePath`
- 若无本地源图，则回退使用 `coverUrl`
- 若仍不可用，则展示默认占位图标

### 3.7 设置页 `SettingsPage`

页面目标：

- 展示环境和平台信息
- 展示服务健康状态
- 管理本地设置与本地历史记录

页面模块：

- 运行环境卡片
- 登录会话卡片
- 服务配置卡片
- 服务健康检查卡片
- 本地数据卡片
- 项目说明入口卡片

当前主操作：

- 查看环境类型
- 查看当前平台
- 查看应用版本
- 查看当前账号、登录模式和会话到期时间
- 修改 `baseUrl`，仅开发 / 测试环境
- 查看服务状态、模型状态、服务版本和服务时间
- 手动刷新服务健康检查结果
- 清空历史记录
- 恢复默认设置
- 退出登录
- 查看关于项目

当前交互规则：

- 正式环境默认隐藏高风险编辑入口
- 服务健康检查失败时仅影响当前卡片，不阻塞设置页其他功能
- 清空历史记录前必须确认
- 恢复默认设置后立即持久化

当前不包含：

- “清除全部缓存”通用入口
- 正式环境下修改服务地址

### 3.8 登录页 `LoginPage`

页面目标：

- 承接账号密码登录和注册
- 对齐后端 `login.html / register.html` 的双栏认证界面
- 展示当前登录模式
- 作为受保护页面前的统一入口

页面模块：

- 介绍主页返回入口
- Material 3 响应式双栏壳层
- 左侧项目说明与能力特征区
- 登录 / 注册模式切换区
- 登录表单区
- 注册规则与密码强度区，仅真实接口模式
- 演示账号提示区，仅开发 / 测试环境
- 记住用户名入口，仅登录模式

当前页面状态：

- `idle`
- `submitting`
- `error`
- `success_notice`

交互规则：

- 登录中禁用重复提交
- 真实接口模式允许在同一卡片内切换登录和注册
- 页面结构拆分为壳层、说明面板和表单卡片，页面容器不直接承载大段展示组件
- 注册成功后不自动登录，而是回到登录模式展示成功提示
- 未授权和登录失败必须展示明确错误信息
- 登录成功后直达主控台 `RealtimeDetectPage`
- 登录模式允许本地记住用户名，下次打开登录页自动回填
- 开发 / 测试环境允许填充演示账号

### 3.9 关于页 `AboutPage`

页面目标：

- 对齐后端 `preview.html` 的公开预览页
- 展示项目定位、架构概览、研究背景和系统价值

页面模块：

- 顶部品牌与登录入口区
- 平台总览 Hero 区
- 系统架构概览区
- 平台与设备能力区
- 研究背景区
- 成活率与调控目标区

交互规则：

- 内容以只读说明和登录跳转为主
- 未登录时提供登录入口，已登录时提供进入平台入口
- 文案必须与当前技术选型、接口边界和页面实现保持一致

## 4. 当前验收要点

当前页面规格验收应满足：

- 所有已接入路由都可正常打开
- 未登录时访问受保护页面会被重定向到登录页
- 单图识别页可稳定完成选图、识别和跳转
- 结果页、历史页、设置页与当前仓储逻辑一致
- 实时识别页明确区分测试链路和真实摄像头链路
- 文档中不再出现“拖拽已支持”“假框已实现”“清除缓存已实现”等与代码不一致的表述

## 5. 结论

页面规格文档必须服务于当前代码，而不是保留早期规划稿。后续每当页面新增能力、删除能力或调整状态流时，都必须同步更新本文档中对应页面章节。
