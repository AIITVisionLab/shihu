# 石斛病虫害识别项目 页面规格与交互文档

## 1. 文档目标

本文档用于定义石斛病虫害识别项目 Flutter 前端各页面的模块结构、交互行为、状态切换、异常处理和跨平台表现，作为页面实现和文档验收的直接依据。

说明：

- 本文档只记录当前代码已经实现或已经明确预留的页面行为。
- 对尚未接入的能力，会明确标注为“占位”或“后续接入”，不把规划项写成已实现项。

## 2. 全局交互规则

### 2.1 路由规则

当前主流程固定为：

- `SplashPage -> LoginPage / HomePage`
- `LoginPage -> HomePage`
- `HomePage -> DetectPage`
- `HomePage -> RealtimeDetectPage`
- `HomePage -> HistoryPage`
- `HomePage -> SettingsPage`
- `DetectPage -> ResultPage`
- `HistoryPage -> ResultPage`
- `SettingsPage -> AboutPage`

### 2.2 全局状态规则

当前核心页面至少覆盖以下状态：

- `loading`
- `success`
- `empty`
- `error`

补充说明：

- `permissionDenied` 当前只作为实时识别页后续预留状态，现阶段页面尚未接入摄像头权限流程。

### 2.3 全局反馈规则

- 点击后应立即给出交互反馈
- 异步动作必须进入加载态或禁用重复提交
- 错误必须展示为明确失败态或错误提示
- 删除、清空等危险操作必须二次确认
- 不允许用假成功值掩盖真实失败

### 2.4 跨平台适配规则

- 手机端优先纵向布局
- Web、Windows、Linux、macOS 优先居中内容区和更宽的卡片布局
- 桌面端补充 hover、focus 和键盘可用性
- Android、iOS、Web、Windows、Linux、macOS、OpenHarmony / 鸿蒙 的入口顺序和主流程保持一致
- 平台差异主要收敛到文件选择、权限、窗口尺寸和运行时判断

## 3. 页面规格

### 3.1 启动页 `SplashPage`

页面目标：

- 完成冷启动初始化
- 加载本地设置、应用版本信息和登录态
- 跳转主入口

页面模块：

- Logo 区
- 项目名称区
- 标语区
- 状态提示区

当前页面状态：

- `loading`：初始化中
- `error`：初始化失败

交互规则：

- 初始化成功后按登录态进入首页或登录页
- 初始化失败时显示错误信息和“重试初始化”按钮
- 启动页不允许白屏

### 3.2 首页 `HomePage`

页面目标：

- 作为主导航入口
- 展示项目定位和当前基线能力

页面模块：

- 标题与版本区
- 基线说明区
- 功能入口卡片区

当前入口：

- 开始识别
- 实时监测
- 历史记录
- 设置

交互规则：

- 入口按钮一屏可见
- 桌面端支持焦点和点击反馈
- 首页文案必须明确当前已实现和待补能力

### 3.3 单图识别页 `DetectPage`

页面目标：

- 选择或拍摄石斛图片
- 预览图片
- 发起单图识别

页面模块：

- 识别说明卡片
- 图片预览区
- 操作区

主操作：

- 从相册选择
- 使用相机
- 清空选择
- 开始识别

当前页面状态：

- `idle`：未选择图片
- `selected`：已选择图片
- `running`：识别中
- `error`：选图失败或识别失败

交互规则：

- 未选择图片时禁用“开始识别”
- 识别中禁用重复选择和重复提交
- 识别成功后跳转结果页
- 失败信息通过 `SnackBar` 展示

当前实现边界：

- 桌面端拖拽入口仅在文案中预留，当前尚未实现
- 当前默认调用正式单图识别接口；开发 / 测试环境可显式切回受控 mock 仓储

### 3.4 实时识别页 `RealtimeDetectPage`

页面目标：

- 为摄像头实时识别链路预留稳定入口

当前实现状态：

- 当前页面已接入预览区、摘要区、检测框元数据区和会话控制区
- 开发 / 测试环境已接入测试帧链路，可刷新摘要和检测框元数据
- 当前不申请摄像头权限
- 当前不展示真实预览画面和真实检测框渲染

后续预留状态：

- `idle`
- `initializing`
- `running`
- `paused`
- `permissionDenied`
- `error`

当前交互规则：

- 用户进入页面后可看到稳定的页面结构和接入边界说明
- 开发 / 测试环境允许手动启动、暂停和继续测试帧会话
- 在未接入真实链路前，不允许伪造预览或假框效果

### 3.5 结果页 `ResultPage`

页面目标：

- 展示一次识别任务的完整结果
- 支持保存到历史记录

页面模块：

- 结果图区域
- 摘要信息区
- 检测列表区
- 防治建议区
- 保存按钮区

当前页面状态：

- `success`
- `empty`
- `error`

交互规则：

- 优先展示摘要结果
- 当 `detections` 为空时展示“未检测到明显异常”
- 从单图识别进入时允许保存
- 从历史记录进入时禁用再次保存

图片展示规则：

- 优先使用当前识别源图片路径
- 其次使用 `annotatedUrl`
- 再次回退到 `originalUrl`

### 3.6 历史记录页 `HistoryPage`

页面目标：

- 展示本地历史记录
- 支持查看详情、删除单条、清空全部

页面模块：

- 顶部操作区
- 历史列表区
- 空状态区

当前页面状态：

- `loading`
- `success`
- `empty`
- `error`

交互规则：

- 点击列表项进入结果页
- 删除单条前二次确认
- 清空全部前二次确认
- 记录为空时展示空状态提示

缩略图规则：

- 优先展示 `sourceImagePath`
- 若无本地源图，则回退使用 `coverUrl`
- 若仍不可用，则展示默认占位图标

### 3.7 设置页 `SettingsPage`

页面目标：

- 展示环境和平台信息
- 展示服务健康状态
- 管理本地设置与本地历史记录

页面模块：

- 运行环境卡片
- 登录会话卡片
- 服务配置卡片
- 服务健康检查卡片
- 本地数据卡片
- 项目说明入口卡片

当前主操作：

- 查看环境类型
- 查看当前平台
- 查看应用版本
- 查看当前账号、登录模式和会话到期时间
- 修改 `baseUrl`，仅开发 / 测试环境
- 查看服务状态、模型状态、服务版本和服务时间
- 手动刷新服务健康检查结果
- 清空历史记录
- 恢复默认设置
- 退出登录
- 查看关于项目

当前交互规则：

- 正式环境默认隐藏高风险编辑入口
- 服务健康检查失败时仅影响当前卡片，不阻塞设置页其他功能
- 清空历史记录前必须确认
- 恢复默认设置后立即持久化

当前不包含：

- “清除全部缓存”通用入口
- 正式环境下修改服务地址

### 3.8 登录页 `LoginPage`

页面目标：

- 承接账号密码登录
- 展示当前登录模式
- 作为受保护页面前的统一入口

页面模块：

- 项目说明区
- 登录表单区
- 演示账号提示区，仅开发 / 测试环境

当前页面状态：

- `idle`
- `submitting`
- `error`

交互规则：

- 登录中禁用重复提交
- 未授权和登录失败必须展示明确错误信息
- 登录成功后进入首页
- 开发 / 测试环境允许填充演示账号

### 3.9 关于页 `AboutPage`

页面目标：

- 展示项目定位、技术基线和开发策略

页面模块：

- 项目定位卡片
- 当前技术基线卡片
- 开发策略卡片

交互规则：

- 内容以只读说明为主
- 文案必须与当前技术选型保持一致

## 4. 当前验收要点

当前页面规格验收应满足：

- 所有已接入路由都可正常打开
- 未登录时访问受保护页面会被重定向到登录页
- 单图识别页可稳定完成选图、识别和跳转
- 结果页、历史页、设置页与当前仓储逻辑一致
- 实时识别页明确区分测试链路和真实摄像头链路
- 文档中不再出现“拖拽已支持”“假框已实现”“清除缓存已实现”等与代码不一致的表述

## 5. 结论

页面规格文档必须服务于当前代码，而不是保留早期规划稿。后续每当页面新增能力、删除能力或调整状态流时，都必须同步更新本文档中对应页面章节。
