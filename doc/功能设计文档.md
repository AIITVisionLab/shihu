# 石斛病虫害识别项目 Flutter 前端功能设计文档

## 1. 文档目标

本文档用于定义石斛病虫害识别项目 Flutter 前端的当前功能范围、页面职责、用户流程和验收边界，作为后续开发、联调和文档同步的功能依据。

本文档重点回答以下问题：

- 当前代码已经实现了哪些功能
- 哪些页面已经可用，哪些只是占位入口
- 用户在各平台上能完成哪些主流程
- 哪些能力仍然处于下一阶段规划

## 2. 产品概述

本项目是一个面向石斛种植与养护场景的跨平台前端应用，围绕石斛叶片病害、虫害和健康状态识别展开。

当前版本重点不是“做一套静态演示稿”，而是先打通一个可运行的前端基线：

- 用户可以登录并恢复上一次会话
- 用户可以选择图片或拍照进入单图识别流程
- 系统可以调用真实单图识别接口并展示结果详情
- 用户可以把结果保存到本地历史记录
- 用户可以查看环境、版本、平台和本地配置

当前版本尚未完成：

- 真实登录接口联调与安全存储加固
- 真实后端识别接口的正式联调与错误策略收口
- 实时识别摄像头链路
- 全平台发布级构建验证

## 3. 当前版本功能范围

### 3.1 已实现功能

- 启动页初始化
- 登录页
- 首页主导航
- 单图识别页
- 结果页
- 历史记录页
- 设置页
- 关于页
- 本地设置持久化
- 本地历史记录持久化

### 3.2 已预留但未完成的功能

- `ApiClient` 网络层封装
- `/api/v1/detect/image`、`/api/v1/detect/realtime/frame` 接口契约
- `/api/v1/auth/login`、`/api/v1/auth/refresh`、`/api/v1/auth/logout` 正式联调
- 平台运行时识别与 `OpenHarmony / 鸿蒙` 编译期平台覆盖

### 3.3 当前不提供的能力

- 假摄像头预览
- 假检测框 overlay
- 远程历史记录接口
- 服务端配置托管

说明：

- 当前版本默认走真实单图识别接口，开发 / 测试环境允许显式切回受控 mock。
- 历史记录和设置页面使用本地真实持久化，不再写成“mock 列表 / mock 设置”。

## 4. 平台与环境范围

### 4.1 目标平台

- Android
- iOS
- macOS
- Web
- Windows
- Linux
- OpenHarmony / 鸿蒙

### 4.2 平台要求

- 各平台主流程入口顺序保持一致
- 页面结构和业务逻辑尽量复用
- 平台差异主要收敛到权限、文件选择、窗口尺寸和运行时判断
- `OpenHarmony / 鸿蒙` 当前以工程接入和运行时适配为主，实际构建依赖本地 OHOS 工具链与签名

### 4.3 环境要求

当前代码只区分三类构建环境：

- `development`
- `staging`
- `production`

其中：

- 开发和测试环境允许调整 `baseUrl`
- 正式环境默认隐藏高风险设置项

## 5. 页面与功能明细

### 5.1 启动页 `SplashPage`

功能目标：

- 展示项目名称和品牌感
- 完成基础初始化
- 自动跳转首页

当前实现：

- 并行初始化本地设置、应用版本信息和登录会话
- 显示加载状态
- 初始化失败时显示错误信息和重试按钮
- 初始化成功后根据登录态跳转首页或登录页

### 5.2 登录页 `LoginPage`

功能目标：

- 承接账号密码登录
- 展示当前登录模式
- 作为受保护页面前的统一入口

当前实现：

- 支持账号和密码输入
- 支持开发 / 测试环境的受控演示登录
- 登录成功后进入首页
- 未授权或登录失败时展示明确错误信息

后续待完成：

- 真实登录接口联调
- 刷新令牌策略收口
- 更安全的会话存储方案

### 5.3 首页 `HomePage`

功能目标：

- 作为整个应用的主入口
- 表达当前技术基线和下一阶段规划
- 提供统一导航

当前实现：

- 展示项目标题、版本和基线说明
- 提供“开始识别 / 实时监测 / 历史记录 / 设置”四个入口
- 显示当前基线已支持的能力摘要

### 5.4 单图识别页 `DetectPage`

功能目标：

- 选择石斛图片或拍照
- 预览图片
- 发起一次单图识别
- 跳转结果页

当前实现：

- 支持从相册选择图片
- 支持相机拍照入口
- 支持图片预览
- 支持识别中状态和失败提示
- 当前默认通过 `DetectRepository -> RealDetectRepository` 调用正式接口
- 开发 / 测试环境可显式切换到 `MockDetectRepository`

后续待完成：

- 与后端契约的正式联调
- 页面级恢复入口与更完整的重试策略

### 5.5 实时识别页 `RealtimeDetectPage`

功能目标：

- 为摄像头实时识别链路保留入口
- 后续承接权限申请、预览和帧识别

当前实现：

- 路由已接入
- 页面已接入正式结构，包含预览区、摘要区、检测框元数据区和会话控制区
- 开发 / 测试环境已接入测试帧轮询链路，可刷新摘要和检测框元数据
- 当前不会申请摄像头权限，不会展示真实摄像头预览或真实叠框渲染

后续待完成：

- 摄像头权限检查
- 预览画面
- 摄像头帧采样与真实实时识别请求
- 把当前检测框元数据渲染为真实 overlay

### 5.6 结果页 `ResultPage`

功能目标：

- 展示一次识别任务的完整结果
- 展示摘要、检测列表、严重程度和防治建议
- 支持把结果保存为历史记录

当前实现：

- 展示结果图或原图回退路径
- 展示主识别结果、置信度、严重程度和健康状态
- 展示检测列表
- 展示防治建议
- 从单图识别进入时允许保存到历史记录
- 从历史记录进入时只读展示，不重复保存

### 5.7 历史记录页 `HistoryPage`

功能目标：

- 浏览本地历史记录
- 查看详情
- 删除单条或清空全部

当前实现：

- 本地加载历史记录
- 空状态提示
- 点击进入结果详情页
- 删除单条记录
- 清空全部历史记录
- 所有危险操作都有二次确认

### 5.8 设置页 `SettingsPage`

功能目标：

- 展示环境、平台、版本和服务配置
- 展示当前服务健康状态
- 管理本地数据

当前实现：

- 展示环境类型、当前平台、应用版本
- 展示当前账号、登录模式和会话到期时间
- 展示 `baseUrl`、连接超时、接收超时
- 展示服务状态、模型状态、服务版本和服务时间
- 支持手动刷新服务健康检查结果
- 仅在开发 / 测试环境开放 `baseUrl` 修改
- 支持清空本地历史记录
- 支持恢复默认设置
- 支持退出登录
- 支持进入关于页

当前不提供：

- 通用“清除缓存”能力
- 正式环境下的高风险配置编辑入口

### 5.9 关于页 `AboutPage`

功能目标：

- 展示项目定位、当前技术基线和开发策略

当前实现：

- 说明项目面向石斛场景
- 说明当前使用的 Flutter、Riverpod、GoRouter、Dio、SharedPreferences、OpenHarmony Flutter 模板等技术基线
- 说明单图识别默认走真实接口、开发 / 测试环境可切回 mock 的策略

## 6. 用户主流程

### 6.1 当前已打通主流程

1. 用户进入启动页
2. 应用初始化本地设置、版本信息和登录会话
3. 未登录用户进入登录页
4. 用户登录成功后进入首页
5. 用户从首页进入单图识别页
6. 用户选择图片或拍照
7. 用户点击“开始识别”
8. 页面进入识别中状态
9. 系统调用单图识别接口并返回标准识别结果
10. 页面跳转到结果页展示详情
11. 用户可将结果保存到本地历史记录
12. 用户可在历史记录页再次打开该结果

### 6.2 当前已打通辅助流程

1. 用户从登录页进入首页
2. 用户从首页进入设置页
3. 查看环境、平台、版本和当前账号信息
4. 在开发 / 测试环境中修改 `baseUrl`
5. 清空历史记录或恢复默认设置
6. 退出登录或进入关于页查看项目说明

### 6.3 当前已打通的测试实时流程

1. 用户从首页进入实时识别页
2. 用户手动启动测试帧链路
3. 页面按会话状态刷新主识别结果、健康状态、单帧耗时和检测框元数据
4. 用户可暂停或继续当前测试会话

## 7. 替身数据与真实接口策略

当前策略如下：

- 登录态在开发 / 测试环境默认使用受控演示仓储，可通过 `USE_MOCK_AUTH=false` 切换到真实接口联调
- 单图识别链路默认走真实接口，开发 / 测试环境允许显式启用受控 mock 以便演示和回归
- 历史记录使用本地真实持久化，不使用假列表替身
- 设置页使用本地真实持久化，不使用假设置对象
- 实时识别页在开发 / 测试环境使用受控测试帧链路，但不使用假摄像头预览或假叠框制造“已实现”错觉

切换原则：

- 通过 `AuthRepository` / `DetectRepository` / `RealtimeDetectRepository` Provider 切换具体实现，不改页面契约
- 正式环境不允许继续保留假结果分支
