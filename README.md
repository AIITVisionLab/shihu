# 石斛病虫害识别项目 Flutter 前端

## 项目说明

这是一个面向石斛种植与养护场景的 Flutter 全平台前端项目，当前仓库同时包含可运行代码和配套设计文档。

目标平台包括：

- Android
- iOS
- macOS
- Web
- Windows
- Linux（rpm包,deb包，AppImage包，二进制文件）
- OpenHarmony / 鸿蒙

目标架构包括:

- X86_64
- ARM64
- LoongArch


平台说明：

- `macOS` 需要在 macOS 主机上完成本地构建与签名验证。
- `OpenHarmony / 鸿蒙` 已纳入工程结构和运行时适配，但实际生成 `hap` 仍依赖对应 Flutter OHOS 工具链和本地签名配置。

## 当前代码基线

当前仓库已经不是“仅文档仓库”，而是具备首轮可运行基线的 Flutter 工程。

已完成：

- Flutter 工程初始化、主题、路由、分析规则和基础目录结构
- `Splash / Home / Detect / Result / History / Settings / About` 主流程页面
- `flutter_riverpod` 状态管理接入
- 单图识别主链路：选图 / 拍照 -> 真实 `/api/v1/detect/image` 识别 -> 结果页 -> 保存本地历史
- 本地设置持久化、本地历史记录持久化
- `macOS` 与 `OpenHarmony / 鸿蒙` 平台枚举、运行时识别和基础工程预留
- 一组基础 Widget 测试与仓储测试

当前未完成：

- `/api/v1/detect/image` 的正式联调、错误码细化与异常重试策略
- 实时识别页的摄像头预览、权限处理和帧推理链路
- 关键集成测试、日志采集和全平台构建回归

## 关键决策摘要

以下约束已经确定，后续实现不要偏离：

- 项目主题固定为 `石斛病虫害识别`，不按泛化农业 Demo 组织。
- 正式环境不允许返回假结果、伪成功值或静默降级。
- V1 首版必须保留 `本地历史记录` 能力。
- 状态管理统一使用 `flutter_riverpod`。
- 单图识别默认走真实接口；开发 / 测试环境可通过 `USE_MOCK_DETECT=true` 切回受控 mock；历史记录和设置始终走本地真实持久化。
- 实时识别页当前仅保留入口和占位说明，不伪造摄像头预览和假框效果。
- 平台差异统一收敛到适配层，不在页面层分叉业务逻辑。
