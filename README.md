## 关键决策摘要

以下约束已经确定，后续实现不要偏离：

- 项目主题明确为 `石斛病虫害识别`，不是泛化农业识别项目。
- 项目按 `生产环境标准` 设计，不按 demo 思路组织。
- 正式环境不允许 `mock`、替身数据、假结果、伪成功值。
- 不允许通过静默降级掩盖问题。
- 不允许通过无必要的 `try/catch` 逃避问题。
- 前端与 Java 后端统一使用 `HTTP / HTTPS + JSON + multipart/form-data`。
- V1 首版必须支持 `本地历史记录`。
- 如果后端提供历史记录接口，允许在 Repository 层切换到远程实现，但页面层不改。
- 状态管理默认推荐 `flutter_riverpod`。
- 多平台页面结构和主流程保持一致，差异收敛到适配层。
