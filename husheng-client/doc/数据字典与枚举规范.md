# 斛生 数据字典与枚举规范

## 1. 文档目标

本文档用于统一当前客户端仍在使用的字段、模型、枚举和值域，避免页面、状态层、本地存储和接口层出现语义偏差。

## 2. 通用规则

- JSON 字段统一使用 `camelCase`
- 代码标识符统一使用英文
- 界面文案统一使用中文
- 时间字段按后端真实契约处理
- 可空字段必须明确声明
- `2026-03-12` 的产品化重构新增了视频中心运行时模型
- 当前版本不再维护旧单图识别页和历史页对应模型

## 3. 核心模型

### 3.1 `ApiResponse<T>`

用途：

- 前端网络层处理通用包裹协议时使用的泛型模型

字段：

- `code`：业务状态码
- `message`：业务提示
- `data`：业务数据，可空

说明：

- 当前 `web` 分支后端的 7 个已接通接口并不统一使用该结构。
- 该模型保留在网络层作为通用工具，不代表当前后端主契约。

### 3.2 `AppSettings`

用途：

- 本地应用设置

字段：

- `baseUrl`：设备服务基础地址
- `connectTimeoutMs`：连接超时，单位毫秒
- `receiveTimeoutMs`：接收超时，单位毫秒
- `enableLog`：是否开启日志
- `buildFlavor`：构建环境

### 3.3 `AuthUser`

用途：

- 当前登录用户信息

字段：

- `userId`：用户 ID
- `account`：登录账号
- `displayName`：显示名称
- `roles`：角色列表，可为空

### 3.4 `AuthSession`

用途：

- 前端本地登录态对象

字段：

- `accessToken`：前端本地兼容字段
- `refreshToken`：可空，本地替身模式可能使用
- `sessionCookie`：服务端会话 Cookie，可空
- `tokenType`：认证类型，真实后端当前为 `Session`
- `expiresAt`：过期时间，可空
- `loginMode`：登录模式
- `user`：当前用户信息

说明：

- 当前真实后端基于 `HttpSession + JSESSIONID`，不是 Bearer Token。
- `accessToken` 不应被解释为后端真实令牌。

### 3.5 `DeviceStatus`

用途：

- `/api/status` 返回的设备状态对象

字段：

- `deviceId`：设备 ID
- `deviceName`：设备名称
- `temperature`：温度值，可空
- `temperatureUnit`：温度单位
- `humidity`：湿度值，可空
- `humidityUnit`：湿度单位
- `light`：光照值，可空
- `lightUnit`：光照单位
- `mq2`：MQ2 值，可空
- `mq2Unit`：MQ2 单位
- `errorCode`：错误码，可空
- `ledOn`：LED 状态，可空
- `updatedAt`：毫秒时间戳

派生语义：

- `alertLevel`：根据错误码映射为 `safe / warning / danger / unknown`
- `canControlLed`：是否满足 LED 控制最小条件

说明：

- 当前版本中的 `DeviceStatus` 已回收为设备状态实体，不再直接承载界面标题、状态说明和指标格式化文案。

### 3.6 `DeviceStatusViewData`

用途：

- 设备状态在界面层使用的展示派生对象

字段：

- `deviceLabel`：设备展示名称
- `alertLevel`：界面使用的运行等级
- `alertTitle`：面向界面的状态标题
- `alertDescription`：面向界面的状态说明
- `freshnessLabel`：数据新鲜度文案
- `ledLabel`：LED 展示文案
- `isFresh`：当前数据是否仍在新鲜窗口内
- `temperatureLabel`：温度展示文案
- `humidityLabel`：湿度展示文案
- `lightLabel`：光照展示文案
- `mq2Label`：MQ2 展示文案

### 3.7 `LedOperationReceipt`

用途：

- `/api/ops/led` 的前端接收对象

字段：

- `status`：后端状态值
- `requestId`：请求编号，可空
- `message`：后端消息

### 3.8 `ServiceHealthInfo`

用途：

- 服务健康检查结果

说明：

- 当前保留为内部联调模型，不在普通用户界面直接展示

字段：

- `status`：服务状态，例如 `up / down / unknown`
- `responseText`：健康接口原始响应文本
- `checkedAt`：客户端完成检查时间

### 3.9 `VideoStreamInfo`

用途：

- `/api/video/streams` 的前端接收对象

字段：

- `streamId`：视频流标识
- `deviceId`：设备标识
- `displayName`：展示名称
- `gatewayPageUrl`：网关页地址
- `playerUrl`：播放页地址
- `preferredMode`：推荐播放模式
- `fallbackMode`：回退播放模式
- `publicHost`：公网主机名或 IP
- `webrtcPort`：WebRTC 端口
- `available`：当前是否在线
- `aiResultForwarded`：当前是否已开启 AI 结果转发

### 3.10 `AiDetectionSummary`

用途：

- `/api/edge/ai-detections/latest` 与 `/api/edge/ai-detections/history` 的前端接收对象

字段：

- `type`：结果类型
- `deviceId`：设备标识
- `stream`：视频流标识
- `timestampMs`：结果时间戳，可空
- `frameId`：帧编号，可空
- `imageWidth / imageHeight`：图像尺寸，可空
- `detectionCount`：目标数量
- `empty`：是否为空结果
- `summary`：后端汇总说明
- `overallRiskLevel`：总体风险等级
- `items`：识别项列表

### 3.11 `AiDetectionItem`

用途：

- AI 汇总结果中的单项检测对象

字段：

- `classId`：类别编号，可空
- `originalClassName`：边缘端原始类别名
- `displayName`：前端展示名称
- `category`：类别归属
- `confidence`：置信度，可空
- `riskLevel`：风险等级
- `advice`：建议动作
- `bbox / quad`：几何信息

### 3.12 `AiDetectionOverview`

用途：

- 值守台 AI 卡片使用的组合结果

字段：

- `latest`：最新结果，可空
- `history`：最近历史列表

### 3.13 `PlatformLogSummary`

用途：

- `/api/logs/summary` 的前端接收对象

字段：

- `count`：累计事件数
- `file`：日志文件路径
- `supportedTypes`：支持的事件类型列表

### 3.14 `PlatformLogEntry`

用途：

- `/api/logs` 的前端接收对象

字段：

- `eventId`：事件编号
- `timestampMs`：事件时间戳，可空
- `type`：事件类型
- `deviceId`：设备标识
- `summary`：事件摘要
- `details`：原始详情，可空

### 3.15 `PlatformLogOverview`

用途：

- 设置页平台日志卡片使用的组合结果

字段：

- `summary`：日志摘要
- `recentEntries`：最近事件列表

### 3.16 `PlatformLogQuery`

用途：

- 设置页平台日志卡片使用的查询条件

字段：

- `keyword`：关键字，可空
- `type`：事件类型，可空
- `limit`：查询条数上限

## 4. 当前枚举

### 4.1 `BuildFlavor`

枚举值：

- `development`
- `staging`
- `production`

### 4.2 `AuthLoginMode`

枚举值：

- `real`
- `mock`

说明：

- `real`：真实后端登录
- `mock`：本地替身登录

### 4.3 `PlatformType`

枚举值：

- `android`
- `ios`
- `macos`
- `web`
- `windows`
- `linux`
- `ohos`

### 4.4 `DeviceAlertLevel`

枚举值：

- `safe`
- `warning`
- `danger`
- `unknown`

## 5. 本地存储键

当前仍在使用的本地存储键：

- `app_settings`
- `auth_session`
- `remembered_account`

说明：

- 当前版本已经删除历史记录相关本地结构，不再维护 `history_records`。

## 6. 文档边界

以下内容不再属于当前数据字典维护范围：

- 单图识别页结果模型
- 旧历史记录模型
- 摄像头取帧、叠框绘制等旧页面专用枚举

如果未来恢复相应业务，必须在对应代码回归后重新补写本文件，而不是保留过时字段说明。
