# 斛生 前后端接口契约文档

## 1. 文档目标

本文档用于说明当前 Flutter 客户端与后端的真实接口契约、会话方式和联调边界。本文档只以当前工作区可见源码为准：

- Flutter 客户端：当前仓库
- Spring Boot 后端：`CjhAIIt/iot-onenet-refact-main`

## 2. 当前已确认接口

当前客户端运行时围绕以下 11 个接口组织页面和状态：

- `POST /api/login`
- `POST /api/register`
- `GET /api/check-login`
- `POST /api/logout`
- `GET /api/status`
- `POST /api/ops/led`
- `GET /api/video/streams`
- `GET /api/edge/ai-detections/latest`
- `GET /api/edge/ai-detections/history`
- `GET /api/logs`
- `GET /api/logs/summary`

说明：

- 登录、设备状态、LED 控制、视频中心、AI 巡检和平台日志当前都以 `iot-onenet-refact-main` 可见源码为准。
- 视频中心继续按 `GET /api/video/streams` 字段接入，并额外消费 `aiResultForwarded` 状态。
- 值守台当前会主动查询 AI 汇总结果，不再停留在“只定义协作边界”阶段。

## 3. 认证契约

### 3.1 会话模式

当前后端使用：

- `HttpSession`
- `JSESSIONID`

前端约束：

- Web 端认证请求启用浏览器凭据，由浏览器自动管理 Cookie
- 非 Web 端从响应头提取 `JSESSIONID`，并保存到本地会话
- 当前后端不要求 `Authorization: Bearer <token>`
- `AuthSession.accessToken` 只是前端本地会话对象中的兼容字段，不代表后端真实 Bearer Token
- 当前默认设备服务地址已经切到公网反向代理入口 `http://101.35.79.76`；如需回到本机联调，可通过 `--dart-define=BASE_URL=http://127.0.0.1:8085` 覆盖，Android / iOS 原生端如需直连 HTTP 服务仍需要额外放开明文访问
- Web 端若采用跨源部署，浏览器对跨站 Cookie 的限制会直接影响登录态建立，部署时应优先使用同源代理或统一 HTTPS

### 3.2 `POST /api/login`

用途：

- 用户登录
- 建立服务端会话

请求体：

```json
{
  "username": "root",
  "password": "12345"
}
```

成功响应示例：

```json
{
  "success": true,
  "message": "登录成功"
}
```

失败响应示例：

```json
{
  "success": false,
  "message": "用户名或密码错误"
}
```

字段说明：

- `username`：登录用户名
- `password`：登录密码
- `success`：是否成功
- `message`：结果说明

### 3.3 `POST /api/register`

用途：

- 创建新账号

请求体：

```json
{
  "username": "new_user",
  "password": "demo123456",
  "confirmPassword": "demo123456"
}
```

成功响应示例：

```json
{
  "success": true,
  "message": "注册成功，请使用新账号登录"
}
```

字段约束：

- `username`：3 到 32 位，只允许字母、数字和下划线
- `password`：6 到 32 位
- `confirmPassword`：需与 `password` 一致

### 3.4 `GET /api/check-login`

用途：

- 恢复本地登录态时检查当前会话是否仍然有效

成功响应示例：

```json
{
  "loggedIn": true,
  "username": "root"
}
```

未登录响应示例：

```json
{
  "loggedIn": false
}
```

字段说明：

- `loggedIn`：当前会话是否有效
- `username`：登录成功时返回

### 3.5 `POST /api/logout`

用途：

- 注销当前会话

成功响应示例：

```json
{
  "success": true,
  "message": "已登出"
}
```

## 4. 设备状态契约

### 4.1 `GET /api/status`

用途：

- 拉取当前设备缓存状态
- 供首页快照、主控台和设置页复用

响应示例：

```json
{
  "deviceId": "dev_001",
  "deviceName": "石斛培育柜",
  "temperature": { "value": 24.5, "unit": "°C" },
  "humidity": { "value": 82.0, "unit": "%" },
  "light": { "value": 1500.0, "unit": "Lux" },
  "mq2": { "value": 18.0, "unit": "ppm" },
  "error": { "value": 0 },
  "led": { "value": true },
  "updatedAt": 1741399200000
}
```

字段说明：

- `deviceId`：设备 ID
- `deviceName`：设备名称
- `temperature / humidity / light / mq2`：包含 `value` 和 `unit`
- `error.value`：错误码
- `led.value`：LED 当前状态
- `updatedAt`：毫秒时间戳

错误码语义：

- `0`：系统运行正常
- `1`：系统预警，建议人工复核
- `2`：系统严重告警
- 其他值：前端按未知状态展示

## 5. LED 控制契约

### 5.1 `POST /api/ops/led`

用途：

- 控制设备 LED 开关

请求体：

```json
{
  "deviceId": "dev_001",
  "deviceName": "石斛培育柜",
  "led": true
}
```

成功响应示例：

```json
{
  "status": "accepted",
  "requestId": "req_001",
  "message": "指令已下发"
}
```

排队响应示例：

```json
{
  "status": "pending",
  "requestId": "req_002",
  "message": "OneNET API调用失败,已登记到待处理队列: ..."
}
```

字段说明：

- `status`：后端处理状态，前端当前接受 `accepted / success / ok / pending`
- `requestId`：请求编号，可空
- `message`：提示信息

## 6. 健康检查契约

### 6.1 `GET /api/health`

用途：

- 检查后端服务是否可用

当前前端兼容两种返回：

1. 字符串 `ok`
2. 对象结构，例如：

```json
{
  "status": "up",
  "message": "ok",
  "checkedAt": "2026-03-10T12:30:00Z"
}
```

前端展示策略：

- 页面直接展示原始响应文本
- 页面记录本次检查完成时间
- 不再擅自推导不存在的扩展字段

说明：

- 当前 `/api/health` 仍保留为内部联调接口，不在普通用户界面直接展示

## 7. 视频中心契约

### 7.1 `GET /api/video/streams`

用途：

- 为视频中心提供全部视频流入口

响应包裹：

- 兼容 `{code,msg,data}` 或 `{code,message,data}` 两种外层结构
- 当前前端同时兼容后端直接返回数组的情况

`data[]` 字段：

- `streamId`
- `deviceId`
- `displayName`
- `gatewayPageUrl`
- `playerUrl`
- `preferredMode`
- `fallbackMode`
- `publicHost`
- `webrtcPort`
- `available`
- `aiResultForwarded`

前端展示策略：

- 直接展示播放地址和网关页地址
- 优先使用 `playerUrl` 作为播放入口
- 展示 `aiResultForwarded` 状态，并提示用户回到值守台查看 AI 巡检结论
- 当业务后端暂未提供该接口且当前运行配置已提供视频网关地址时，客户端回退为单路默认视频入口，避免视频页整体不可用

## 8. AI 巡检契约

### 8.1 `GET /api/edge/ai-detections/latest`

用途：

- 拉取当前最新一条 AI 检测汇总结果
- 供值守台展示最新结论、风险等级和重点目标

响应包裹：

- 固定使用 `{code,msg,data}`
- 成功时 `code=0`
- 当前没有最新结果时，`data` 允许为空

`data` 字段：

- `type`
- `deviceId`
- `stream`
- `timestampMs`
- `frameId`
- `imageWidth`
- `imageHeight`
- `detectionCount`
- `empty`
- `summary`
- `overallRiskLevel`
- `items[]`

`items[]` 字段：

- `classId`
- `originalClassName`
- `displayName`
- `category`
- `confidence`
- `riskLevel`
- `advice`
- `bbox`
- `quad`

### 8.2 `GET /api/edge/ai-detections/history`

用途：

- 拉取最近几条 AI 检测历史
- 供值守台展示“最近记录”

查询参数：

- `limit`：返回条数，默认 `20`

前端展示策略：

- 最新卡片优先展示 `summary`、`overallRiskLevel` 和 `items[]`
- 最近记录区直接展示后端已归纳过的汇总文案，不回退到边缘端原始 JSON
- 当前仍不恢复单图识别页、叠框渲染和本地历史页

## 9. 平台日志契约

### 9.1 `GET /api/logs/summary`

用途：

- 拉取平台日志摘要
- 供设置页展示累计事件数、日志文件位置和支持类型

响应包裹：

- 固定使用 `{code,msg,data}`
- 成功时 `code=0`

`data` 字段：

- `count`
- `file`
- `supportedTypes[]`

### 9.2 `GET /api/logs`

用途：

- 拉取最近平台日志
- 供设置页展示最近设备上报、下发回写和 AI 事件
- 供设置页按关键字、类型和条数范围重新查询

查询参数：

- `type`：可空
- `keyword`：可空
- `limit`：默认 `100`

`data[]` 字段：

- `eventId`
- `timestampMs`
- `type`
- `deviceId`
- `summary`
- `details`

前端展示策略：

- 设置页当前默认展示最近 6 条事件
- 设置页支持把 `type`、`keyword` 和 `limit` 作为筛选条件直接回传给后端
- 优先展示 `summary` 和事件类型，不把原始详情 JSON 直接当正文展开

## 10. 客户端本地替身能力

以下能力存在于 Flutter 代码中，但不属于后端契约：

- `MockAuthRepository`

说明：

- 该能力只用于开发 / 测试环境本地联调
- 需要显式传入 `USE_MOCK_AUTH=true`
- 正式环境不允许走本地替身认证

## 11. 当前不属于契约的接口

以下路径不属于当前交付契约：

- `/api/v1/detect/image`
- `/api/v1/detect/realtime/frame`
- `/api/video/streams/{streamId}`
- `/api/edge/ai-detections`

说明：

- `/api/video/streams/{streamId}` 当前前端未直接使用。
- `POST /api/edge/ai-detections` 属于 RK3568 到 Java 后端的上送接口，不属于当前前端直接请求范围。
