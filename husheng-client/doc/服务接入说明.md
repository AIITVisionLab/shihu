# 斛生 服务接入说明

## 1. 当前结论

当前 Flutter 客户端应接入：

- 业务后端：`CjhAIIt/iot-onenet-refact-main`
- 边缘/视频网关：`AIITVisionLab/shihu`

职责边界固定为：

- `iot-onenet-refact-main` 提供登录、注册、会话检查、设备状态、LED 控制、AI 查询、日志查询、视频流信息查询
- `shihu` 负责边缘采集、AI 上送、`go2rtc/frp` 视频网关，不作为 Flutter 直接依赖的业务后端

也就是说，Flutter 不直接把 `shihu` 当主业务服务来连，而是通过 Java 业务后端消费它提供的边缘协作能力。

## 2. 当前默认地址

当前仓库默认业务服务地址已经改为：

- `http://101.35.79.76`

视频默认仍按当前公网网关地址：

- `http://101.35.79.76:1984`

这套默认值对应“真实服务已经部署在服务器上”的接入方式，适合直接连接现网。

## 3. 历史地址迁移

为了避免旧安装包或旧本地设置继续指向已废弃地址，当前版本会把以下历史默认地址自动迁移到当前环境默认值：

- `http://101.35.79.76:19081`
- `http://101.35.79.76:8082`
- `http://127.0.0.1:8082`
- `http://127.0.0.1:8085`

如果用户自己明确改成了其他地址，客户端不会强行覆盖。

## 4. 本地联调方式

如果后面需要回到本机启动 Java 服务联调，可以显式覆盖：

```bash
flutter run --dart-define=BASE_URL=http://127.0.0.1:8085
```

设置页里也可以直接把“设备服务地址”改回本地地址。

## 5. 当前接入风险

当前开发环境已经验证：

- `http://101.35.79.76/api/health` 可以返回 `ok`
- `http://101.35.79.76/api/status` 可以返回设备状态 JSON
- `http://101.35.79.76/api/video/streams` 可以返回视频流 JSON
- `POST http://101.35.79.76/api/login` + `GET /api/check-login` 可以建立并验证会话

同时也确认：

- `http://101.35.79.76:19081` 对外直连时不会返回标准 HTTP 响应

因此当前客户端必须优先走公网反向代理入口，而不是直接使用 `19081`。

## 6. 客户端模块归属

当前仓库已经把服务接入相关逻辑收口到独立模块：

- `lib/features/service_config/domain/service_endpoint_resolver.dart`
- `lib/features/service_config/application/service_config_providers.dart`
- `lib/features/service_config/infrastructure/service_health_repository.dart`

这样服务地址规范化、旧地址迁移、实际命中端点和健康检查不再散落在 `core`、`settings`、`auth` 等多个位置。
