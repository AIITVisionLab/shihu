# K230 与腾讯云协同配置说明

## 目标与拓扑

当前视频链路固定为：

```text
K230 --RTSP(H264)--> RK3568 go2rtc --WebRTC/MSE/HLS--> 浏览器
                                 \
                                  +--frpc--> 腾讯云 frps --> 公网
```

当前补充机制：
- 当前临时回退到 Wi-Fi 单路径
- RK3568 初始从 `rtsp://192.168.1.103:8554/test` 拉流
- 前端和公网入口地址不变，底层 RTSP 源地址会在 Wi-Fi 网段内自动切换

## K230 侧要求

K230 侧需要满足：
- 主码流为 `H264`
- 能稳定输出 `rtsp://<当前Wi-Fi-IP>:8554/test`
- 持续向 RK3568 的 `POST /api/ai` 上送 `AI_DETECTIONS`
- 当前默认走 Wi-Fi，推荐保持 `192.168.1.103`
- 如果 Wi-Fi 地址变化，RK3568 允许在 `192.168.1.0/24` 网段内自动切换

K230 不需要主动修改 RK3568 配置，也不需要通知 RK3568 切换流地址；只要 `/api/ai` 和 RTSP 持续正常，RK3568 会按当前 Wi-Fi 来源地址收敛。

## RK3568 侧要求

RK3568 当前负责三件事：
- `go2rtc` 拉取 K230 RTSP
- `edgelink-gateway` 接收 `/api/ai`
- 在 Wi-Fi 网段内根据 `/api/ai` 来源地址自动改写 `config/go2rtc.yaml`

当前重新启用了自动源地址切换：
- `video_source_switch.enabled = true`
- `allowed_source_cidrs = 192.168.1.0/24`
- `/api/ai` 继续用于 AI 结果接收和转发

## 腾讯云侧要求

腾讯云继续保持当前 frp 方案：
- `frps` 提供公网入口
- 公网页面：`http://101.35.79.76:1984/`
- 公网播放页：`http://101.35.79.76:1984/stream.html?src=k230&mode=webrtc,mse`
- WebRTC 端口：`101.35.79.76:8555`

自动源地址切换不要求腾讯云修改任何配置。

## 联调顺序

1. 先确认 K230 当前走的是 `lan` 还是 `wifi`
2. 确认 K230 能持续输出 RTSP 和 `/api/ai`
3. 确认 RK3568 本地 `go2rtc` 页面能播放 `k230`
4. 确认 RK3568 本地 `go2rtc` 能持续从当前 Wi-Fi 地址拉流
5. 最后再确认公网入口播放仍正常
