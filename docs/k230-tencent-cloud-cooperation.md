# K230 与腾讯云协同配置说明

## 目标与拓扑

当前视频链路固定为：

```text
K230 --RTSP(H264)--> RK3568 go2rtc --WebRTC/MSE/HLS--> 浏览器
                                 \
                                  +--frpc--> 腾讯云 frps --> 公网
```

当前补充机制：
- K230 采用“有线优先、Wi-Fi 兜底”接入局域网
- RK3568 根据 `POST /api/ai` 的来源地址自动更新 `go2rtc` 的 `streams.k230`
- 前端和公网入口地址不变，只是底层 RTSP 源地址会在局域网内切换

## K230 侧要求

K230 侧需要满足：
- 主码流为 `H264`
- 能稳定输出 `rtsp://<当前IP>:8554/test`
- 持续向 RK3568 的 `POST /api/ai` 上送 `AI_DETECTIONS`
- 有线可用时优先使用 `172.18.8.103`
- 有线不可用时允许回退到 Wi-Fi

K230 不需要主动修改 RK3568 配置，也不需要通知 RK3568 切换流地址；只要 `/api/ai` 和 RTSP 持续正常，RK3568 会自动收敛。

## RK3568 侧要求

RK3568 当前负责三件事：
- `go2rtc` 拉取 K230 RTSP
- `edgelink-gateway` 接收 `/api/ai`
- 在来源地址变化时自动改写 `config/go2rtc.yaml` 并重启 `go2rtc`

自动切换判断依据：
- 只认 `deviceId == "k230"` 的 `/api/ai` 请求
- 只认请求来源 IP，不靠 ARP 或扫描猜测
- 同一新 IP 连续命中达到阈值后才切换
- 切换后播放入口不变

## 腾讯云侧要求

腾讯云继续保持当前 frp 方案：
- `frps` 提供公网入口
- 公网页面：`http://101.35.79.76:1984/`
- 公网播放页：`http://101.35.79.76:1984/stream.html?src=k230&mode=webrtc,mse`
- WebRTC 端口：`101.35.79.76:8555`

自动源地址切换不要求腾讯云修改任何配置。

## 联调顺序

1. 先确认 K230 当前走的是 `lan` 还是 `wifi`
2. 确认 K230 能持续输出 RTSP 和 `/api/ai`
3. 确认 RK3568 本地 `go2rtc` 页面能播放 `k230`
4. 触发 K230 从有线切到 Wi-Fi，观察 RK3568 是否自动切换源地址
5. 最后再确认公网入口播放仍正常
